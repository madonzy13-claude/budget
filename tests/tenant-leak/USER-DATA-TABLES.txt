# USER-DATA-TABLES.txt
# Authoritative enumeration of user-data tables for tenant-leak CI gate (Plan 10).
#
# Format: <schema.table>  <SCOPE>  # comment
#
# SCOPE values:
#   TENANT-SCOPED  - RLS policy keys off app.tenant_ids (uuid array)
#   USER-SCOPED    - RLS policy keys off app.current_user_id (NOT app.tenant_ids)
#   EXCLUDED       - Infrastructure table; intentionally outside RLS (see Pitfall 10)
#
# Tests 1 + 4 use this file at runtime. Parse rules:
#   - Lines starting with # are comments — skip
#   - Blank lines — skip
#   - Token 1: schema.table
#   - Token 2: TENANT-SCOPED | USER-SCOPED | EXCLUDED
#
# PC-12: USER-SCOPED tables use a different RLS predicate (app.current_user_id),
#        so test 1 (no-GUC zero rows) asserts both GUCs absent → 0 rows, and also
#        runs a cross-user filter sub-test using app.current_user_id only.
# T-13 (green-washing): to validate this gate is real, flip app_role to BYPASSRLS in
#        post-migration.sql and rerun — every TENANT-SCOPED and USER-SCOPED test must fail.
