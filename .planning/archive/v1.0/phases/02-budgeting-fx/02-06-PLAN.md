<objective>
Ship the **expense_ledger writer** — the single transactional path that mints every EXPENSE / INCOME / TRANSFER row. ALTER expense_ledger to add Phase-2 columns (transaction_date, note, account_id, category_id, kind, transfer_group_id, note_tsv) per D-05-b/f and DROP corrected_by_id per D-05-a. The transaction-repo write opens ONE `withTenantTx` and atomically: (1) INSERTs ledger row(s), (2) UPDATEs accounts.current_balance, (3) UPSERTs spending_by_category_month projection (ENGR-14), (4) calls writeOutbox `budgeting.transaction.created` (Pitfall 7), (5) optional writeAudit. Application layer fetches FX via FrankfurterFxProvider, validates currency allowlist + deposit-rate freshness (D-02-d 60-minute window), checks workspace_share_dirty flag (D-02-c).

The TransactionRepo ships TWO entry points: `create(rows, userId, tenantId)` opens its own withTenantTx; `createInTx(tx, rows, userId, tenantId)` accepts a caller-managed tx so plan 02-08's confirm-recurring-draft can run the ledger insert + draft UPDATE in a single withTenantTx. Both are introduced in this plan; plan 02-08 only consumes them.

The transaction-capture-form's currency picker sources its options exclusively from the `budgeting.supported_currencies` allowlist via a `listSupportedCurrencies()` server action. The picker mirrors the pattern shipped by plan 02-04 Task 3 for the account-form currency picker. Server-side validation (already in `createTransaction`) rejects any code outside the allowlist, but the UI picker prevents the bad path entirely — a user cannot select 'XYZ' unless it has been seeded into `budgeting.supported_currencies`.

Purpose: This is the heart of Phase 2. Plans 02-04 (accounts) and 02-05 (categories) deliver the lookup tables. This plan delivers the _write path_ that consumes them. Plans 02-07 (corrections), 02-08 (recurring), 02-09 (search/bulk) all extend this writer rather than rebuild it.

Output: A bullet-proof ledger writer (with both create() and createInTx() entry points); transaction capture form per UI-SPEC with allowlist-bound currency picker; e2e for create-transaction golden path; ledger immutability invariant test that asserts permission-denied at SQL.
</objective>
