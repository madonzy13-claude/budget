  cadence: Cadence;

Cadence rules per D-PH2-03 (validate at domain level + DB CHECK):

- DAILY: no extras required
- WEEKLY: weeklyDow required (0..6)
- MONTHLY: anchorDay required (1..31; clamped to last-of-month per existing month-end logic Jan 31 → Feb 28)
- YEARLY: yearlyMonth required (1..12) AND anchorDay required (1..31, clamped to that month's length per leap-year)

DAILY case: `return prev.add({ days: 1 })`
YEARLY case (RESEARCH §Code Examples line 641):

```typescript
const targetYear = prev.year + 1;
const targetMonth = spec.yearlyMonth!;
const daysInTarget = Temporal.PlainDate.from({
  year: targetYear,
  month: targetMonth,
  day: 1,
}).daysInMonth;
const targetDay = Math.min(spec.anchorDay!, daysInTarget);
return Temporal.PlainDate.from({
  year: targetYear,
  month: targetMonth,
  day: targetDay,
});
```
