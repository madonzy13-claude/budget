- DAILY: no extras required
- WEEKLY: weeklyDow required (0..6)
- MONTHLY: anchorDay required (1..31; clamped to last-of-month per existing month-end logic Jan 31 → Feb 28)
- YEARLY: yearlyMonth required (1..12) AND anchorDay required (1..31, clamped to that month's length per leap-year)

   amount_original_cents, currency_original,
   amount_converted_cents, fx_rate, fx_as_of,

Simplest path: at draft INSERT time, call `fxProvider.rateAsOf(rule.currency, budget.currency, dueDate)` inside the worker — same pattern as create-transaction app service. For Phase 2 this is acceptable; profile in Phase 5 if slow.
