<objective>
Rewrite the reserve-mutating application use-cases for the new model. Adjustments become pure signed deltas (decision E). Wallet balance/type changes set ONLY the userDefined amount — they no longer allocate reserve to categories (decision C: internal is derived ΣR, never written). Category deletion honors both modes (decision J). The RESERVE_TOPUP task becomes a single budget-level surplus reconcile (emit iff surplus ≠ 0). All greedy paths (`applyExpectedChange`/`applyWalletDelta`/`applyExclude`/`setReserveActualMany`) are removed from these use-cases.

Purpose: With the engine + orchestrator in place (05-09..05-12), the mutation layer must STOP doing greedy allocation and stored-actual bookkeeping. The only durable reserve write is the append-only adjustment delta; everything else is recomputed. This plan makes the writes match the replay model and drives the surplus task off the orchestrator.
Output: rewritten adjust/wallet/archive/disable use-cases + the surplus reconcile + `categories-repo` cleanup (remove `setReserveActualMany`); their tests updated to the new behavior. Project rule: every changed use-case keeps/gains a test; RESERVE_TOPUP direction tests stay green.
</objective>
