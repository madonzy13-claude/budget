LIVE current point (D-04): import computeBudgetWealthNow (11-03) → {capitalization_cents, investment_value_cents, currency}; append as the newest point at now() so the rightmost bucket reflects the last 3h live.

Bucket aggregation (D-20): rangeBuckets(from,to) monthly default; daily for this-month/≤62d. Aggregate snapshots to the bucket by taking the LAST snapshot in each bucket (month-end / day-end sample) — document the choice (last-in-bucket = end-of-period value, matches a value time-series).

Stats:

- grow/loss (D-15): start = first bucket value, end = last bucket value (incl. live point); delta_cents = end − start; delta_pct = start===0 ? null : Number(delta)×100/Number(start).
- month-over-month dynamics (D-16): for consecutive monthly samples, pct[i] = (v[i]−v[i-1])×100/v[i-1] (skip when v[i-1]===0).
- monthly_avg_grow (D-16): simple mean of the dynamics pct array (one-to-one with the bars).

Per-type pie (D-18, investments view): Σ value per holding_type over non-archived holdings, FX→default_ccy. Reuse the 11-03 holdings valuation; group by holding_type. Returns {holding_type, value_cents}[] — the FE maps holding_type → Phase-9 UI_TYPE_COLOR.

FX + tx + DI + route: FxProvider.rateAsOf; withTenantTxRead; boot.ts closure DI; X-Budget-ID; Zod ?from,?to,?view ∈ {capitalization,investments}; bigint→string at route.
