<threat_model>

## Trust Boundaries

| Boundary                               | Description                                                          |
| -------------------------------------- | -------------------------------------------------------------------- |
| push payload → lock screen             | Notification body is visible without auth — must carry no financials |
| push notification url → app navigation | Deep-link target must be a validated /budgets/<id>/<tab> shape       |
| ?task= search param → banner expansion | Untrusted query param drives only client-side row expansion          |
| browser permission prompt → subscribe  | User-gesture-gated; subscription endpoint stored server-side         |

## STRIDE Threat Register

| Threat ID  | Category                  | Component               | Disposition | Mitigation Plan                                                                                                                                                                                              |
| ---------- | ------------------------- | ----------------------- | ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| T-08-05-01 | Info Disclosure           | push payload            | mitigate    | D-15: NOTIFICATION_TYPES bodies are generic strings (no amounts, no category names); test asserts body contains no financial digits (ASVS V3)                                                                |
| T-08-05-02 | Tampering / open-redirect | notificationclick url   | mitigate    | url is constructed server-side from the fixed `/budgets/${budgetId}/${tab}?task=${taskId}` template with tab from the registry (fixed set), not from arbitrary payload; SW opens only same-origin clients    |
| T-08-05-03 | DoS                       | stale push subscription | mitigate    | 410/404 from push endpoint deletes the subscription record (RESEARCH Pitfall 4) so dead endpoints don't accumulate                                                                                           |
| T-08-05-04 | Spoofing                  | permission/subscribe    | accept      | beforeinstallprompt + Notification.requestPermission are user-gesture-gated browser APIs; no server auth surface added                                                                                       |
| T-08-05-05 | Tampering                 | ?task= deep-link param  | mitigate    | The param only selects which already-authorized pending task to expand; an unknown/foreign id matches nothing and lands silently (D-14) — it grants no access and triggers no fetch of another tenant's task |

</threat_model>
