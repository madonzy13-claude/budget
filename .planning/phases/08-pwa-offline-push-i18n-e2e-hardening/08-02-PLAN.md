<threat_model>

## Trust Boundaries

| Boundary            | Description                                                         |
| ------------------- | ------------------------------------------------------------------- |
| browser → /push/\*  | Untrusted subscription endpoint + key material crosses into storage |
| tenant A ↔ tenant B | Subscription lookup for dispatch must not cross tenants             |

## STRIDE Threat Register

| Threat ID  | Category         | Component                 | Disposition | Mitigation Plan                                                                                                                    |
| ---------- | ---------------- | ------------------------- | ----------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| T-08-02-01 | Spoofing         | /push/subscribe           | mitigate    | Session guard (`c.get("session")` → 401) on every state-changing handler; userId taken from session, never request body (ASVS V2)  |
| T-08-02-02 | Info Disclosure  | getSubscriptionsForBudget | mitigate    | Query runs under tenant GUC; RLS from 08-01 confines rows to the dispatching tenant (ASVS V4)                                      |
| T-08-02-03 | Input Validation | subscribe/prefs payloads  | mitigate    | zValidator on POST/PATCH; endpoint is z.string().url(); notificationType is a fixed enum (ASVS V5)                                 |
| T-08-02-04 | Tampering        | outbox emission           | mitigate    | writeOutbox runs in the same tx as the task INSERT; gated to real inserts only so replay/idempotent no-ops do not double-fire push |

</threat_model>
