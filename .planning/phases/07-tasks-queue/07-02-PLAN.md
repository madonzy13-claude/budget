the `resolve-task.ts` application service. This is the ports/adapters foundation
every Wave 2+ generator and the new POST resolve route depends on.

DO NOTHING for emit), Pattern A (`withTenantTx` for all RLS-scoped writes). All
new methods accept an OPTIONAL `tx` parameter so callers inside an existing
`withTenantTx` can piggyback their writes onto the same transaction — critical
for atomic auto-resolve hooks in Plan 04/05/06.

(Plan 07) to call into the same TaskRepo. Centralizing the write surface here
prevents drift across the 11 mutation sites that will call emit/resolve.

Plan 01 (reserve-topup, confirm-draft, resolve-idempotency) gain their first
