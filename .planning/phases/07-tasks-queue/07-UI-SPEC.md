# Phase 7 — UI Design Contract: Tasks Queue

> Visual and interaction contract for enabling the task banner action buttons, wiring
> the Reserves Actions column, and adding the cushion-target-months field to Settings
> and Onboarding. Generated by gsd-ui-researcher; verified by gsd-ui-checker.
>
> **Source-of-truth precedence:** `DESIGN.md` > `apps/web/src/app/global.css` >
> Phase 3–6 UI-SPEC carry-forward > this document.
>
> **Carry-forward locked (do NOT redefine):** dark-only theme, 4-multiple spacing
> scale, Inter (`--font-sans`) + IBM Plex Sans (`--font-numeric`) stacks, single-
> yellow-accent scarcity rule, `--info-ring` focus ring, BDP sticky pill tabs,
> `<Sheet>` primitive, `<AlertDialog>` primitive, `sonner` toast, TaskBanner DOM
> shape (Phase 3), TaskBannerRow DOM shape (Phase 3 — only the action button
> changes), autosave-per-field + blur-to-save + toast pattern (Phase 4/5/6).
>
> **Phase 7 adds no new design primitives.** It activates disabled interactive
> elements, adds one numeric input, and adds one live-preview text line. Every
> visual token is inherited.

## Spacing Scale

Declared values (multiples of 4 — DESIGN.md `spacing` tokens, carry-forward from Phase 6):

| Token | Value | Usage                                                                           |
| ----- | ----- | ------------------------------------------------------------------------------- |
| xxs   | 4px   | Icon gaps, chip padding, badge internal padding                                 |
| xs    | 8px   | Compact element spacing, gap between kind badge and action button in banner row |
| sm    | 12px  | Input internal padding (vertical), gap between preview line and months input    |
| md    | 16px  | Default element spacing, banner row horizontal padding, form field gaps         |
| lg    | 24px  | Section padding, cushion-section internal gap between fields                    |
| xl    | 32px  | Layout column gaps                                                              |
| xxl   | 48px  | Major section breaks                                                            |

Exceptions:

- Banner row height: fixed 48px (h-12) — established in Phase 3, unchanged.
- Touch targets: action buttons in banner rows minimum 32px height (size="sm"); meets
  44px tap target via surrounding row height.
- Reserves Actions column icon button: 32×32px minimum tap target (p-1 + icon-16).
- `sm` = 12px declared exception: 12px is Tailwind `p-3`, matching the established
  input padding used in Phase 5 and Phase 6 forms. 8px (xs) is too tight for vertical
  input padding and the preview-line gap; 16px (md) is too wide. This 12px usage is
  a declared exception to the standard 4/8/16/24/32/48/64 set, consistent with prior
  phases.

## Typography

All roles carry forward from DESIGN.md via Phase 3–6 UI-SPECs. Declared for Phase 7
surfaces only:

| Role               | Size | Weight | Line Height | Font                           | Usage                                                |
| ------------------ | ---- | ------ | ----------- | ------------------------------ | ---------------------------------------------------- |
| Body (banner)      | 14px | 400    | 1.5         | Inter (--font-sans)            | Task title in banner row (`text-sm`)                 |
| Label (kind badge) | 12px | 400    | 1.4         | Inter (--font-sans)            | Kind chip text inside `<Badge variant="secondary">`  |
| Button (action)    | 14px | 600    | 1           | Inter (--font-sans)            | Action button label in banner row                    |
| Label (input)      | 14px | 400    | 1.4         | Inter (--font-sans)            | "Target months" field label in Settings + Onboarding |

Two weights in use: regular (400) for body copy, labels, captions, and badges;
semibold (600) for button labels only. Preview numbers render in `--font-numeric`
(IBM Plex Sans, tabular-nums) per the project rule: all financial figures use the
numeric font stack.

## Component Inventory

### Modified in Phase 7

#### 1. `TaskBannerRow` (`apps/web/src/components/budgeting/task-banner-row.tsx`)

**Change:** Drop `disabled`, `aria-disabled="true"`, and `title={t("bdp.tasks.actionComingSoon")}` from the `<Button>`. Wire `onClick` per kind. Update `TaskKind` union to remove `STALE_WALLET` and `MONTH_END_REVIEW`, add `CUSHION_BELOW_TARGET`.

**DOM shape (unchanged from Phase 3):**

```
div[role="listitem"] h-12 flex items-center gap-3 border-b px-4
  span.flex-1.truncate.text-sm   ← task title (i18n, no raw payload)
  Badge[variant="secondary"]     ← kind chip
  Button[variant="primary" size="sm"]  ← action button (NOW ENABLED)
```

**Action routing per kind:**

- `RESERVE_TOPUP`: `router.push(`/budgets/${budgetId}/reserves?task=${task.id}`)` — navigates away.
- `CUSHION_BELOW_TARGET`: `router.push(`/budgets/${budgetId}/wallets?task=${task.id}#cushion`)` — navigates away.
- `CONFIRM_DRAFT`: inline `clientApiFetch` to `POST /recurring-rules/.../confirm`. On success: optimistically collapse row (remove from local list). On error: show sonner toast (`t("bdp.tasks.confirmError")`).

**Loading state (CONFIRM_DRAFT only):**

- Button shows spinner (Lucide `Loader2` 14px, `animate-spin`) while in-flight.
- Button is `disabled` during the request to prevent double-submit.
- No loading state for deep-link kinds (navigation is instant).

**Interaction invariant:** Task title and kind badge are never interactive. Only the action button fires events. Payload fields never render as raw DOM text (Phase 3 T-03-06-03 invariant, carry-forward).

#### 2. `TaskKind` type union (`task-banner-row.tsx` + shared types)

**Old:** `"RESERVE_TOPUP" | "CONFIRM_DRAFT" | "STALE_WALLET" | "MONTH_END_REVIEW"`
**New:** `"RESERVE_TOPUP" | "CONFIRM_DRAFT" | "CUSHION_BELOW_TARGET"`

Remove `bdp.tasks.actionComingSoon` i18n key from all three message catalogs.

#### 3. `ReservesTableRow` (`apps/web/src/components/budgeting/reserves-tab/reserves-table-row.tsx`)

**Change:** Wire the Phase 5 D-PH5-R6 Actions column placeholder to an actionable icon button.

**Actions column spec:**

- Icon: Lucide `PencilLine` 16px, color `--muted` at rest, `--body-on-dark` on hover.
- Button: `variant="ghost"` `size="icon"` (32×32px) — no label, `aria-label={t("reserves.actions.editBalance")}`.
- Visible only on rows where the category contributes to a PENDING `RESERVE_TOPUP` task. When no pending task: column cell renders empty (no icon, same cell width).
- Click behavior: opens the existing inline reserve-balance edit path (same as the existing pen-icon mechanism — no new sheet or modal).

#### 4. `CushionSection` (`apps/web/src/components/settings/cushion-section.tsx`)

**Change:** Add `cushion_target_months` numeric input + live shortfall preview line between the master toggle and the display-mode toggle.

**Placement order (top to bottom):**

1. Master toggle `cushion_enabled` (existing).
2. `cushion_target_months` input (NEW) — hidden (`display: none`) when `cushion_enabled = false`.
3. Live shortfall preview line (NEW) — hidden when `cushion_enabled = false`.
4. Display-mode toggle `cushion_mode_enabled` (existing).

**Months input spec:**

- `<Input type="number" min="1" max="60" step="1">` — HTML5 native constraints.
- Label: `t("settings.cushion.targetMonthsLabel")` — 14px / weight 400.
- Default value on mount: `cushion_target_months` from server (default 6 when column just added).
- Save trigger: blur (autosave pattern, same as Phase 5/6 wallet cells). PATCH `/budgets/:id` body `{cushion_target_months: value}`.
- Validation: integer 1–60. If user types out-of-range: show inline error below input in `--trading-down` color, 12px / weight 400: `t("settings.cushion.targetMonthsError")`. Block PATCH until valid.
- Success feedback: sonner toast (`t("settings.cushion.saved")`), same pattern as Phase 6 Settings fields.
- Width: `w-24` (96px) — short numeric field, left-aligned, not full-width.

**Live shortfall preview line spec:**

- Rendered as a `<p>` below the months input, 12px / IBM Plex Sans (--font-numeric) / weight 400 / line-height 1.4.
- Source: `GET /budgets/:id/cushion-summary` — polled on mount and after each saved months change.
- Format: `t("settings.cushion.preview", { actual, required, shortfall })`
  - EN example: `Current: €3,200 / €6,000 (−€2,800)`
  - ICU placeholders: `{actual}`, `{required}`, `{shortfall}` — all formatted via `Intl.NumberFormat` with budget currency.
- Color:
  - `shortfall > 0`: text `--trading-down` (#f6465d) — below target.
  - `shortfall <= 0`: text `--trading-up` (#0ecb81) — target met.
- Loading state: skeleton shimmer (`animate-pulse bg-[--surface-elevated-dark] h-4 w-48 rounded`) while fetch in-flight.
- Error state: `t("settings.cushion.previewError")` in `--muted` color — graceful, non-blocking.

#### 5. Onboarding Cushion Step (`apps/web/src/components/onboarding/`)

**Change:** Add the same `cushion_target_months` numeric input to the existing cushion step, placed immediately below the master `cushion_enabled` toggle.

**Spec:** Identical to the Settings field above with these differences:

- No live shortfall preview line (onboarding has no real data yet).
- No blur-to-save — value is collected as wizard form state, submitted on "Finish".
- Default: `6` pre-filled.
- Hidden when master toggle is off (same CSS `display: none` gating).
- Validation: same integer 1–60 with inline error.

#### 6. `CategorySlider` (`apps/web/src/components/budgeting/category-slider.tsx`)

**Change:** Add silent cushion-mirror linked state. No new visual elements.

**Behavior (invisible to user):**

- `linked` is plain `useState<boolean>` initialized to `initialCushion == null || initialCushion === initialPlanned`.
- On planned field change: if `linked`, call `form.setValue("cushionCents", newValue)` in same React batch.
- On cushion field change: call `setLinked(false)` — silent break.
- No chain icon, no broken-chain icon, no re-link affordance rendered.
- Cushion input hidden when `cushion_enabled = false` — existing gating unchanged.

### Unchanged in Phase 7

- `TaskBanner` (`task-banner.tsx`) — count chip, expand/collapse, 60s poll, visibility invalidation, ARIA accordion. Untouched.
- All BDP tab navigation, sticky tab bar, pull-to-refresh shell.
- Reserves table layout (Phase 5).

## Accessibility

All carry-forward from Phase 3–6. Phase 7 additions:

- Action button `aria-label` for deep-link kinds (since clicking navigates away):
  - `RESERVE_TOPUP`: `aria-label={t("bdp.tasks.action.RESERVE_TOPUP.ariaLabel")}` — EN: `"Go to Reserves to fix top-up"`

- Preview line: `aria-live="polite"` so screen readers announce when shortfall value changes after saving months.
