Frankfurter adapter is the EXACT structural analog (packages/budgeting/src/adapters/fx/frankfurter.ts):

- constructor(private cache: <CacheRepo>, private fetchFn: typeof fetch = fetch) — inject fetchFn so tests mock HTTP without network.
- cache-then-live: lookup cache; on miss, fetchFn(`https://<fixed-host>/...`); if !r.ok throw new Error(`<provider> http ${r.status}`); parse; convert number->string at the boundary (ACL); upsert cache; return.
- catch { fallback to most-recent cached; if none throw a typed NoPriceAvailable error }.

Provider endpoints (09-RESEARCH.md Price Provider API Research):

- twelve_data: GET https://api.twelvedata.com/price?symbol=<SYM>&apikey=<TWELVE_DATA_API_KEY> -> { price: "<num>" }; currency from instruments.quote_currency. Batch up to 8 comma-separated symbols when called for many.
- coingecko: GET https://api.coingecko.com/api/v3/simple/price?ids=<id>&vs_currencies=usd with header x-cg-demo-api-key=<COINGECKO_API_KEY> -> { "<id>": { "usd": <num> } }; currency USD.
- metals_dev: GET https://api.metals.dev/v1/latest?api_key=<METALS_DEV_API_KEY>&currency=USD&unit=troy_ounce. THROW MetalsDailyOnlyError if invoked with an hourly-context flag (Pitfall 3).
  Keys come from env (loadEnv / process.env) read in the factory (P06) and injected; adapters take the key via constructor, never read process.env directly.

InstrumentRepo.search SQL (09-RESEARCH.md Code Examples — local trigram, NEVER a provider):
SELECT id, symbol, display_name, asset_class, quote_currency FROM budgeting.instruments
WHERE active = true AND (symbol ILIKE $1 || '%' OR display_name ILIKE '%' || $1 || '%')
ORDER BY CASE WHEN symbol ILIKE $1 THEN 0 WHEN symbol ILIKE $1 || '%' THEN 1 ELSE 2 END, display_name LIMIT 20;
Min query length >= 2 (D-07 / Claude discretion). Parameterized — pass the query as an sql binding, never string-concatenated.

Drizzle adapter analogs:

- HoldingRepo: pattern packages/budgeting/src/adapters/persistence/task-repo.ts (withTenantTx for tenant-scoped writes, sql tagged templates, bigint cents at boundary). Methods: create, update, archive (set archived_at), listForBudget (active only, enriched), reorder (sort_order), findById.
- InstrumentRepo + PriceCacheRepo: reference-data, use withInfraTx (worker_role) for writes, plain pool read for search. upsert(instrumentId, price, currency) ON CONFLICT (instrument_id) DO UPDATE.

Money at the adapter boundary ONLY — cents (bigint) <-> Money conversion happens in the repo enrich step (list read computes value/PL via P02 portfolio-metrics + FxProvider rate), never in the domain entity.
