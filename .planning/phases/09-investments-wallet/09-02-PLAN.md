class Holding fields: id, tenantId, name, holdingType, group (string|null), instrumentId (string|null; null=custom), buyPriceCents (bigint), buyCurrency, quantity (string; big.js crypto-grade), currentPriceCents (bigint), currentPriceCurrency, sortOrder, archivedAt (Date|null), createdAt

Money / big.js (packages/shared-kernel/src/money.ts) — already the project money tool; reuse for all arithmetic. Cents are bigint in the entity; quantity is a big.js-precision string.

FX rule (INV-08, D-13): if currentPriceCurrency !== buyCurrency, the CALLER converts currentPrice into buyCurrency using a rate (from FxProvider.rateAsOf) and passes it in. portfolio-metrics functions take a plain rate (string) argument; they do NOT import FxProvider (keeps domain pure).

Weight rules (INV-10, D-13):

- Each holding's value is converted to the BUDGET DEFAULT currency for the weight denominator (caller passes a perHoldingBudgetCcyValue or a rate map).
- Within-group weight = holdingBudgetCcyValue / groupBudgetCcyTotal; whole-portfolio weight (ungrouped) = holdingBudgetCcyValue / portfolioBudgetCcyTotal.
- Group-% = groupBudgetCcyTotal / portfolioBudgetCcyTotal.
- Cash holdings (cash_fx): value = amount (no quantity x price); no P/L (return null sentinel for P/L).
- real_estate / other (custom): value = currentPrice x quantity (quantity defaults to 1 per D-32); manual P/L.

NO "pending"/"stale" anywhere (A2) — these functions assume a current price is always present (the API blocks save otherwise). Delisted does not change the math (last cached price used).
