## Typography

Phase 3 uses **3 type roles + 1 numeric role** drawn from DESIGN.md type ramp and pre-defined utilities in `global.css` (L184–255). No new sizes invented.

| Role    | Token                         | Size | Weight | Line Height | Font stack                                                             | Phase 3 usage                                                                                                                       |
| ------- | ----------------------------- | ---- | ------ | ----------- | ---------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| Body    | `body-md`                     | 14px | 400    | 1.5         | Inter (`--font-sans`)                                                  | Default running text: dropdown row names, task-row titles, card body strip, nav-link labels                                         |
| Title   | `title-sm` (`.text-title-sm`) | 16px | 600    | 1.4         | Inter                                                                  | Card header (budget name); active pill label; current-budget name in top-nav trigger; trust-badge precedent DESIGN.md L426          |
| Display | `title-lg` (`.text-title-lg`) | 24px | 600    | 1.3         | Inter                                                                  | Home `<h1>` ("My budgets" or i18n equivalent); BDP placeholder-page `<h1>`                                                          |
| Numeric | `num-md` (`.text-num-md`)     | 16px | 500    | 1.4         | IBM Plex Sans (`--font-numeric`), `font-variant-numeric: tabular-nums` | All currency values on home cards (current-month spent, wallets total, overspent amounts); currency-code badge inside switcher rows |

**Weights declared: exactly 2 — 400 (body) and 600 (title/display).** Numeric role uses weight 500 per DESIGN.md `number-md` token L84 — a single tabular-numerals exception that is not a new editorial weight, it ships in `global.css` already.

**Body line-height:** 1.5 (DESIGN.md `body-md`). **Title/display line-height:** 1.3–1.4 (DESIGN.md title-sm / title-lg).

**Don't:**

- Don't introduce `display-md` / `display-sm` (40/32px) on home — those are marketing roles. Home is a product surface.
- Don't soften display weight to 400 — DESIGN.md L604 "Don't soften display weight."
- Don't render any currency value in Inter — every number goes through `.num` or `.text-num-md` (IBM Plex Sans + tabular nums) per DESIGN.md L594.

| User action                              | Expected response                                                                                                  | Timing           |

### 2. `BudgetSwitcher` Popover — NAV-01, NAV-02, NAV-04

**Trigger (in top-nav):**

- Shape: text button + chevron, no border, hover background `--surface-elevated-dark`, rounded `--radius-md` (6px).
- Content order: `[Lock | Users icon]` private/shared glyph (lucide `Lock` / `Users` per CONTEXT discretion) — gap 4px — budget name in `.text-title-sm` (16px / 600) — gap 4px — `ChevronDown` lucide (16px stroke).
- Active budget name truncates with ellipsis at viewport-dependent character cap (~24ch desktop, ~12ch mobile).
- Padding: 12px × 16px.

**Popover panel (Radix `Popover` primitive from `ui/popover.tsx`):**

- Background `--popover` (= `--surface-card-dark`), rounded `--radius-lg` (8px), 1px hairline `--border`.
- Min-width: **256px** desktop; viewport-width minus 32px on mobile (Radix Popover collision handling).
- Max-height: `60vh` with vertical scroll inside the panel (D-PH3-05 fallback for >10 budgets).
- Internal layout — two **labeled sections** stacked:
  1. Label `t('nav.switcher.personal')` ("Personal" EN) in `.text-caption` (12px / 500, `--muted-foreground`, uppercase, 8px tracking) — padding `spacing.sm spacing.md` (12px × 16px).
  2. List of `BudgetRow` (see below) for `budget.kind === 'PRIVATE'`.
  3. Label `t('nav.switcher.shared')` ("Shared" EN) — same caption styling.
  4. List of `BudgetRow` for `budget.kind === 'SHARED'`.
  - 1px `--border` divider between sections.

**`BudgetRow` row anatomy:**

- Height: **40px** (matches Radix `DropdownMenuItem` precedent).
- Padding: 8px × 16px.
- Row layout: `[Check icon | empty 16px slot]` — gap 8px — `[Lock | Users]` private/shared glyph (16px) — gap 8px — budget name in `body-md` (14px / 400) — flexible spacer — currency `Badge variant="outline"` with 3-letter monospace code (`USD`, `PLN`, `UAH`) in `.num` (CONTEXT discretion).
- Active row shows a leading lucide `Check` glyph in `--body-on-dark` (D-PH3-06; NEVER yellow per DESIGN.md scarcity).
- Hover: background `--surface-elevated-dark`.
- Focus-visible: `--info-ring` 2px outline at -2px offset.
- a11y: rows use `role="menuitemradio"` + `aria-checked` per Radix dropdown-menu convention (D-PH3-06).

**Interaction contracts:**

| User action                                | Expected response                                                                              | Timing                                           |
| ------------------------------------------ | ---------------------------------------------------------------------------------------------- | ------------------------------------------------ |
| Click switcher trigger                     | Popover opens anchored under trigger with 4px gap; first row focused                           | ≤100ms                                           |
| Click `BudgetRow` for budget X             | `router.push('/budgets/[X.id]/spendings')`; popover closes; new top-nav trigger shows X's name | ≤300ms navigation; popover close ≤100ms (NAV-04) |
| Arrow Down / Arrow Up inside popover       | Focus moves between rows (Radix native); skips section labels                                  | ≤16ms                                            |
| `Escape` key                               | Popover closes; focus returns to trigger                                                       | ≤100ms                                           |
| Click outside popover                      | Popover closes; focus returns to trigger                                                       | ≤100ms                                           |
| Click `Check`-marked (active) row          | Popover closes (no navigation; route doesn't change)                                           | ≤100ms                                           |
| Resize viewport < 768px while popover open | Popover snaps to viewport-width minus 32px margins (Radix native)                              | ≤16ms                                            |

**Empty state (zero budgets — only possible immediately after account delete):** Popover renders a single centered paragraph `t('nav.switcher.empty')` = "No budgets yet. Use + to create one." in `.text-caption` `--muted-foreground`, then a small `Button variant="link" size="sm"` "Create budget" routing to `/budgets/new`. (Edge case; primary empty-state surface for new users is the home page hero — see §3.)

**Mobile pattern:** Same Popover, no Sheet (D-PH3-08). Existing `workspace-switcher.tsx` Sheet code is **deleted** in this phase per CONTEXT §"Routing & legacy cleanup".

**Sticky behavior (D-PH3-01):**

- **Single sticky wrapper** holds task banner + pill tabs row per "Specifics" §Sticky behavior.

### 6. Task Banner Shell — BDP-03

**Visual states:**

| State                 | Treatment                                                                                                                                |
| --------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| Hidden (count = 0)    | Entire banner unmounted (D-PH3-14). Sticky wrapper height = 48px (pills only).                                                           |
| Collapsed (count ≥ 1) | Single 48px row visible above pill tabs row.                                                                                             |
| Expanded              | Collapsed row + inline accordion below with task list. Single sticky wrapper grows; pills stay pinned at the bottom edge of the wrapper. |

**Collapsed row anatomy:**

```
┌──────────────────────────────────────────────────────────────────────┐
│  [⚠ icon]  3 tasks pending                          [▼ ChevronDown]   │
└──────────────────────────────────────────────────────────────────────┘
```

- Background `--surface-card-dark`, full-width, 48px height, padding `spacing.md` horizontal (16px).
- Leading: lucide `AlertCircle` 18px in `--primary` (yellow) — **the only place yellow appears in the banner shell**. Justified per BDP-04 + DESIGN.md scarcity: tasks ARE a primary moment.
- Count chip: rendered as a `Badge variant="default"` (yellow + black) with `t('bdp.tasks.count', {n})` ("{n} tasks pending"). Singular: "1 task pending" via i18n plural rules. **The chip wording uses brand voltage by design — count is the call-to-attention.**
- Trailing: lucide `ChevronDown` 16px `--muted-foreground` rotating 180° on expand.
- Hover background: `--surface-elevated-dark`.
- 1px `--hairline-dark` below banner row when expanded; no hairline when collapsed (banner is a single continuous block with the pills row underneath).

**Expanded accordion anatomy:**

```
┌──────────────────────────────────────────────────────────────────────┐
│  [⚠ icon]  3 tasks pending                          [▲ ChevronUp]    │
│  ─────────────────────────────────────────────────────────────────── │
│  Top up reserve by 25 PLN     [Badge: Reserve]    [Button disabled]  │
│  Confirm draft for Netflix    [Badge: Draft]      [Button disabled]  │
│  Update wallet "Revolut"      [Badge: Stale]      [Button disabled]  │
└──────────────────────────────────────────────────────────────────────┘
```

- Accordion uses `details/summary` or a controlled disclosure (no Radix `Accordion` needed for a single block).
- Animation: `height` transition 200ms `ease-out`.
- Task rows: 48px height each, padding `spacing.md` horizontal, `--surface-card-dark` background, 1px `--hairline-dark` row dividers.
- Row layout (left→right): title `body-md` + flexible spacer + `Badge variant="secondary"` for kind chip + 12px gap + `Button variant="primary" size="sm" disabled` with `aria-disabled="true"` (D-PH3-16).
- Disabled action button shows i18n tooltip `t('bdp.tasks.actionComingSoon')` = "Coming in Phase 7".
- Max-height when expanded: `40vh` with internal `overflow-y: auto`. Mouse-wheel inside list scrolls list; outside scrolls page.

**Data fetch contract (D-PH3-13):**

- **Initial render:** RSC fetches `GET /budgets/[id]/tasks?status=pending` server-side; banner mounts already showing real count (or hidden if 0).
- **Client poll:** React Query `useQuery({ queryKey: ['tasks', budgetId, 'pending'], refetchInterval: 60_000 })`.
- **Visibility pause:** `refetchIntervalInBackground: false` + a `visibilitychange` listener that calls `queryClient.invalidateQueries(...)` on tab re-visible to refresh immediately.
- **Stale tolerance:** up to 60 seconds. Acceptable per CONTEXT D-PH3-13 (Phase 7 generators are cron-driven).

**Interaction contracts:**

| User action                              | Expected response                                                                                                                                                                         | Timing                                   |
| ---------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------- |

| Initial paint with count = 0             | Banner entirely absent from DOM; pill row sits directly under top-nav at `top: 64px`                                                                                                      | RSC                                      |

| Click expanded row's trigger again       | Collapse: reverse animation                                                                                                                                                               | 200ms ease-out                           |

| Tab into expanded list                   | Focus moves to first task-row's disabled action button; `aria-disabled` announces "coming in Phase 7"                                                                                     | Native                                   |
| Background poll (60s tick)               | If count changes: badge text updates; sticky height stays unless transitioning from 0→1 or 1+→0 (which adds/removes the entire row — accept this layout shift, it's rare and unambiguous) | Async; no animation on count text change |
| Switch from foreground to background tab | Poll pauses                                                                                                                                                                               | ≤16ms                                    |
| Switch back to foreground                | Immediate invalidation + refetch                                                                                                                                                          | ≤500ms                                   |

## Motion

Phase 3 declares **three timing tiers** mapped to DESIGN.md's flat-surface philosophy (no atmospheric motion).

| Tier     | Duration          | Easing                                       | Use in Phase 3                                                                                                                                          |
| -------- | ----------------- | -------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Instant  | ≤16ms (one frame) | linear                                       | Hover background swaps; pill text color swap; focus ring appearance; sticky CSS-only pinning                                                            |
| Quick    | ≤100ms            | `cubic-bezier(0.16, 1, 0.3, 1)` (`ease-out`) | Popover open/close; tooltip reveal; dropdown row focus highlight                                                                                        |
| Standard | 200ms             | `cubic-bezier(0.16, 1, 0.3, 1)`              | Task banner expand/collapse height animation; card hover scale (1.00 → 1.01); pill background swap on activation (subtle so route swap feels confirmed) |

**Forbidden:**

- No fade-to-black transitions on route swaps (Next.js default page transitions stay off; RSC streaming is the visual rhythm).
- No spring physics.
- No glassmorphism / atmospheric backdrops (DESIGN.md L605).
- No transform-based fancy entrances on cards — they appear via Suspense streaming, which is the motion contract.

**Reduced-motion support:**

All animations honor `@media (prefers-reduced-motion: reduce)` — durations clamp to 0ms, transitions become instant state swaps. Sticky positioning is CSS-only and unaffected.

## Accessibility Contract

- All interactive elements reachable via Tab; focus order specified in §5 BDP Frame.
- `:focus-visible` ring uses `--info-ring` (`#3b82f6`) at 50% alpha per `global.css` L168 — single source of truth.
- Color contrast on text (verified against tokens):
  - `--body-on-dark` (`#eaecef`) on `--canvas-dark` (`#0b0e11`) = **14.7:1** (WCAG AAA).
  - `--muted-foreground` (`#707a8a`) on `--canvas-dark` = **4.6:1** (WCAG AA for body 14px).
  - `--on-primary` (`#181a20`) on `--primary` (`#fcd535`) = **14.3:1** (WCAG AAA) — active pill text on yellow.
- Switcher dropdown: rows are `role="menuitemradio"` with `aria-checked`; active row's `Check` icon is `aria-hidden` (state already announced).
- BDP pills: rendered as `<a>` links inside a `nav[aria-label={t('bdp.tabs.aria')}]` region; active pill carries `aria-current="page"`.
- Task banner: trigger is a `<button>` with `aria-expanded` and `aria-controls` pointing to the accordion panel id.
- Mobile icon-only pills: each pill carries `aria-label={t('bdp.tab.${kind}')}` so the icon is fully accessible.
