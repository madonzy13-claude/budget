<fresh_user_fixture_decision>

## BLOCKER #14/#15 resolution — Better Auth signUpEmail with cookie copy

Direct INSERT into `identity.users` (the v1 plan approach) bypasses Better Auth's password hashing + the required `identity.account` row, so the seeded user CANNOT subsequently sign in. The existing `apps/web/e2e/cross-tenant-cache.spec.ts` works around this by relying on pre-seeded fixture users (alice/bob) created during compose-stack startup — that's fine for static fixtures but not for the per-scenario fresh user the Gherkin migration playbook (STATE.md quick-task 260507-m3x) mandates.

**Decision: programmatic Better Auth signup.** The fixture imports Better Auth's server-side `auth` instance (or calls the running API's `/auth/sign-up/email` endpoint via fetch when running against a remote stack), creates the user, parses the Set-Cookie header from the response, and seeds the cookie into the Playwright browser context via `context.addCookies(...)`. After that, the browser is "signed in" without ever submitting the sign-in form.

This is the same pattern Better Auth's official docs recommend for E2E test setup: https://www.better-auth.com/docs/integrations/playwright

Budget seeding (creating an organization/budget that the test user owns) happens AFTER signup by calling the running API's `POST /budgets` endpoint with the freshly-acquired session cookie. This composes with the existing `auth.api.createOrganization` plumbing (verified at apps/api/src/routes/budgets.ts:63).
</fresh_user_fixture_decision>

    /** Parse a Set-Cookie header into Playwright's addCookies() shape. */
    function parseSetCookieToPlaywright(setCookieLine: string, urlForDomain: string): { name: string; value: string; domain: string; path: string; httpOnly?: boolean; secure?: boolean; sameSite?: "Strict"|"Lax"|"None" } | null {
      const parts = setCookieLine.split(";").map((s) => s.trim());
      const [nv, ...attrs] = parts;
      const eq = nv.indexOf("=");
      if (eq < 0) return null;
      const name = nv.slice(0, eq);
      const value = nv.slice(eq + 1);
      const url = new URL(urlForDomain);
      let path = "/";
      let httpOnly = false;
      let secure = false;
      let sameSite: "Strict"|"Lax"|"None"|undefined = undefined;
      for (const attr of attrs) {
        const [k, v] = attr.split("=").map((s) => s.trim());
        const lk = k.toLowerCase();
        if (lk === "path") path = v ?? "/";
        else if (lk === "httponly") httpOnly = true;
        else if (lk === "secure") secure = true;
        else if (lk === "samesite") sameSite = (v as any);
      }
      return { name, value, domain: url.hostname, path, httpOnly, secure, sameSite };
    }

        throw new Error(`createBudget failed (${res.status}): ${body}`);
      }
      const body = (await res.json()) as { id: string };
      return body.id;
    }
