<objective>
Fill the placeholder `/budgets/new` route with the 5-step onboarding wizard: name ‚Üí
currency ‚Üí type ‚Üí starter categories ‚Üí review. Persist progress in `onboarding_progress`
so it is resumable, force-redirect users with an incomplete wizard, and retire the stray
`/onboarding` route.

Purpose: ONBD-01..09. Per D-05 it is a single-page React step machine (no per-step
URLs); per D-06 the budget row is created at step 1 and PATCHed onward; per D-08 an
incomplete wizard force-redirects. RESEARCH Q3 RESOLVED: finish redirects to
`/budgets/[id]/spendings` (ROADMAP wins over the UI-SPEC's `/budgets/[id]`).
RESEARCH Pitfall 7: step-1 INSERT must supply a locale-guessed `default_currency`
(NOT NULL column).

ONBD-01 FIX (checker WARNING 6): the layout guard alone does NOT satisfy ONBD-01.
ONBD-01 requires that, after signup, the user is redirected to `/budgets/new`. The
layout guard (Task 2) only redirects users who HAVE an incomplete `onboarding_progress`
row ‚Äî and a fresh user has no row at signup. Task 3 closes this: the Better Auth
post-signup `user.create.after` hook inserts an `onboarding_progress` row
(`step=1, completed_at=null`) for every new user. With that row present, the Task 2
layout guard fires on the user's first authenticated request and redirects them into the
wizard ‚Äî genuinely triggering ONBD-01 for fresh users.

No-row safety (checker WARNING 4): the Task 2 layout guard MUST exit early when a user
has NO `onboarding_progress` row. Existing users created before this feature have no row
and must not be trapped in the wizard. Only an existing row with `completed_at === null`
triggers the redirect.

Output: working wizard; layout guard (with no-row early exit); post-signup row insert;
/onboarding redirect; component tests green.
ONBD-07 (the onboarding_progress backend) ships in Plan 06-04 ‚Äî this plan consumes it.
</objective>
