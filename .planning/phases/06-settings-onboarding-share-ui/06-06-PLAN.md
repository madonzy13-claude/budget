<threat_model>

## Trust Boundaries

| Boundary                | Description                                                       |
| ----------------------- | ----------------------------------------------------------------- |
| browser → API           | Wizard creates/patches a budget and writes onboarding_progress    |
| middleware/layout → API | The redirect guard reads onboarding_progress for the session user |

## STRIDE Threat Register

| Threat ID  | Category               | Component                                                 | Disposition | Mitigation Plan                                                                                                                          |
| ---------- | ---------------------- | --------------------------------------------------------- | ----------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| T-06-06-01 | Information Disclosure | Wizard reads/writes another user's onboarding_progress    | mitigate    | The onboarding endpoints (Plan 06-04) key strictly on the session user; the wizard sends no user_id                                      |
| T-06-06-02 | Tampering              | `?step` query param forged to skip required steps         | mitigate    | `?step` only restores UI position; each "Next" still performs the real API write — a forged step cannot fabricate a budget or categories |
| T-06-06-03 | Denial of Service      | Force-redirect loop traps the user on /budgets/new        | mitigate    | The layout guard skips the redirect when already on `/budgets/new` and when no progress row exists                                       |
| T-06-06-04 | Elevation of Privilege | Budget created during onboarding with a forged kind/owner | mitigate    | `POST /budgets` derives the owner from the session; `kind` is validated by the existing createSchema enum                                |

</threat_model>
