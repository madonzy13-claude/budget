## STRIDE Threat Register

| Threat ID  | Category               | Component                                                           | Disposition | Mitigation Plan                                                                                                        |
| ---------- | ---------------------- | ------------------------------------------------------------------- | ----------- | ---------------------------------------------------------------------------------------------------------------------- |
| T-06-04-01 | Elevation of Privilege | Non-owner archives or deletes a shared budget                       | mitigate    | Owner-only gate (`withBootstrapUserContext` role lookup → 403) on both archive and delete                              |
| T-06-04-02 | Tampering              | Typed-name delete confirmation bypassed by hitting the API directly | mitigate    | Server re-validates `confirmName === budget.name` → 422 `name_mismatch`; client disable is cosmetic only               |
| T-06-04-03 | Information Disclosure | One user reads/writes another user's onboarding_progress            | mitigate    | Endpoints key on `session.user.id`; body never carries a user_id; table RLS keys on `app.current_user_id` (Plan 06-01) |
| T-06-04-04 | Tampering              | SQL injection in archive/delete/onboarding queries                  | mitigate    | Drizzle `sql` template tags with bind params only                                                                      |
| T-06-04-05 | Denial of Service      | Hard delete leaves orphaned child rows / FK errors                  | mitigate    | Rely on `ON DELETE CASCADE` FKs; if any FK is RESTRICT, delete children in the same tx                                 |

</threat_model>
