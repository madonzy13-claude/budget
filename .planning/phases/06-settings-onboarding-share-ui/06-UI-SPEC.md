# Phase 6 — UI Design Contract: Settings, Onboarding & Share UI

> Visual and interaction contract for three settings-shaped form flows. Generated by gsd-ui-researcher; verified by gsd-ui-checker.
>
> **Source-of-truth precedence:** `DESIGN.md` > `apps/web/src/app/global.css` > Phase 3–5 UI-SPEC carry-forward > this document. Every token, color, font, and spacing value below is already defined upstream — this spec describes how they compose into the new surfaces.
>
> **Carry-forward locked (do NOT redefine):** dark-only theme, 4-multiple spacing scale, Inter (`--font-sans`) + IBM Plex Sans (`--font-numeric`) stacks, single-yellow-accent scarcity rule, `--info-ring` focus ring, BDP sticky pill tabs at `top: 64px`, `<Sheet>` primitive, `<AlertDialog>` primitive, `sonner` toast, autosave-per-field + blur-to-save + toast pattern (Phase 4/5).

## Spacing Scale

Declared values (multiples of 4 — DESIGN.md `spacing` tokens):

| Token   | Value | Usage                                                                             |
| ------- | ----- | --------------------------------------------------------------------------------- |
| xxs     | 4px   | Icon gaps, chip padding, inline tight spacing                                     |
| xs      | 8px   | Compact element spacing, accordion trigger icon gap                               |
| sm      | 12px  | Input internal padding (vertical), accordion content inset                        |
| md      | 16px  | Default element spacing, form field gaps, section internal padding                |
| lg      | 24px  | Section padding, card padding, stepper gap                                        |
| xl      | 32px  | Layout column gaps, wizard card padding                                           |
| xxl     | 48px  | Major section breaks between accordion panels when expanded                       |
| section | 80px  | Not used in these three surfaces (settings/wizard/join are form-density surfaces) |

Exceptions:

- Touch targets for icon-only actions (revoke member, copy button): minimum 44px height enforced via `min-h-[44px]`.
- Stepper segment gaps: 8px (xs) between segments; segment width is proportional to `(container - 4 × 8px gap) / 5`.
- Wizard card max-width: 480px centered on mobile-first; no horizontal padding compression below 320px.

## Typography

All roles use Inter (`--font-sans`). IBM Plex Sans (`--font-numeric`) applies only to currency amounts and number inputs.

| Role    | Size | Weight | Line Height | Usage                                                                               |
| ------- | ---- | ------ | ----------- | ----------------------------------------------------------------------------------- |
| Body    | 14px | 400    | 1.5         | Form labels, help text, accordion body copy, member rows                            |
| Label   | 14px | 600    | 1.4         | Field labels above inputs, section headings inside accordion content                |
| Heading | 16px | 600    | 1.4         | Accordion trigger text, wizard step title, join page budget name                    |
| Display | 20px | 600    | 1.35        | Page-level heading ("Budget Settings", "Create your budget", "You've been invited") |

Numeric role (IBM Plex Sans, tabular-nums):

- Currency amounts, member count badge: 14px / weight 500 / line-height 1.4 — reuse `--font-numeric` class from Phase 4.

Two weights only: 400 (regular) + 600 (semibold). No 500, 700, or other weights on text elements (numbers use 500 via numeric role).

## Interaction Model

### Settings Page — `/budgets/[id]/settings`

**Layout:** Single scrollable column. 5 accordion sections, one open at a time (Radix `type="single"`). Default open: "Budget Identity".

**Section order (top to bottom):**

1. Budget Identity
2. Cushion Mode
3. Recurring Rules
4. Members (hidden entirely for PRIVATE budgets — not collapsed, not shown)
5. Danger Zone

**Budget Identity fields (D-01):**

- Click field → input becomes editable (no edit icon needed — cursor-pointer + subtle hover bg `--surface-elevated-dark`).
- Blur → PATCH → sonner toast `"Budget name saved"` / `"Currency updated"`.
- Currency field: editable until first transaction. When locked: field renders as static text + `<Tooltip>` on the lock icon reading `"Currency locked after first transaction"`. Backend signals `has_transactions: boolean` on the budget detail response.
- No Save button anywhere in the section.

**Cushion Mode toggle (D-02):**

- `<Switch>` primitive (Radix — add `switch.tsx` to Phase 6 new components if not present; check `apps/web/src/components/ui/` — not in current list, add it).
- Flip → PATCH `cushion_mode_enabled` → sonner toast `"Cushion mode on"` / `"Cushion mode off"`.
- No confirm dialog. Fully reversible.
- Toggle label: "Cushion mode" with body-sm help text: "Uses the tighter cushion budget limit for all categories."

**Recurring Rules section (D-03):**

- `<RecurringRulesList>` rendered inline inside accordion content.
- "+Add rule" dashed button (neutral — NOT yellow) opens `<Sheet>` containing `<RecurringRuleForm>`.
- Edit (pencil icon on hover) → opens same Sheet pre-populated.
- Delete rule → instant + undo toast (same pattern as Phase 4).

**Members section (D-14, D-16):**

- Visible only when `budget.type === 'SHARED'`. For PRIVATE: section is absent (not rendered).
- Empty state (no non-owner members yet): `"No other members yet."` body text + "Generate share link" button.
- Member row: avatar + name + role badge + "Revoke" ghost button (destructive red text, 44px min-height).
- Revoke → `<AlertDialog>` confirm: `"Remove [name] from this budget? They will lose access immediately."` / Confirm: `"Remove"` (destructive) / Cancel: `"Cancel"`.
- Generate share link: button → POST to generate → reveals `<ShareUrlField>` (read-only input + Copy icon button). Dismissable. Ephemeral — gone on reload. No link history shown.
- Copy button: click → `navigator.clipboard.writeText()` → sonner toast `"Link copied to clipboard"`.
- 7-day TTL displayed as caption below the URL field: `"Expires in 7 days"`.

**Danger Zone section (D-09–D-13):**

- Section heading in destructive red (`--trading-down`). Separated from Members by `<Separator>`.
- Owner sees: Archive button + Delete button.
- Non-owner member sees: Leave budget button only.
- Last owner of SHARED budget: Leave button is disabled + tooltip `"You're the last owner. Delete the budget instead."`.
- Archive: ghost button with red border. Click → `<AlertDialog>`: `"Archive this budget? It will be hidden from your dashboard. Your data is kept."` / Confirm: `"Archive"` / Cancel: `"Keep it"`.
- Delete: filled destructive button. Click → `<AlertDialog>` with typed-name confirmation input. Confirm button enabled only when typed value exactly matches budget name. Confirm label: `"Delete forever"`.
- Leave: ghost button with red border (for non-owners). Click → `<AlertDialog>`: `"Leave [budget name]? You will lose access immediately."` / Confirm: `"Leave"` / Cancel: `"Stay"`.

### Onboarding Wizard — `/budgets/new`

**Layout:** Full-viewport centered column. Max-width 480px. `<WizardLayout>` renders stepper above card, action row pinned to bottom of card (not viewport — scrolls with content on mobile).

**Stepper (D-07):** 5 equal segments. Visual states:

- Completed step: filled `--surface-elevated-dark` + checkmark icon (lucide `Check`, 12px).
- Current step: filled `--primary` (#fcd535) + step number text in `--on-primary` (#181a20).
- Upcoming step: outlined `--hairline-on-dark`, step number in `--muted`.

**Step 1 — Budget Name:**

- Single `<Input>` for budget name.
- Placeholder: `"e.g. Family Budget 2026"`.
- On "Next": INSERT budget row (with locale-guessed default currency e.g. USD) → create `onboarding_progress(user_id, step=1, completed_at)`. Budget type defaults to PRIVATE.
- Validation: name required, max 80 chars. Inline error below field: `"Budget name is required."`.

**Step 2 — Currency:**

- Currency picker (reuse `display-currency-picker` pattern). Default pre-selected from browser locale.
- Help text: `"You can change this later until you add your first transaction."`.
- On "Next": PATCH budget currency + upsert `onboarding_progress(step=2)`.

**Step 3 — Budget Type:**

- Two option cards (not radio buttons — full card selectable). Selected card: yellow left border + yellow ring.
- Option A: "Personal" (lock icon) — `"Just for you"` subtext.
- Option B: "Shared" (users icon) — `"Invite family or household members"` subtext.
- On "Next": PATCH `budget.type` + upsert `onboarding_progress(step=3)`.

**Step 4 — Starter Categories:**

- Fixed list: Housing, Groceries, Transport, Eating Out, Entertainment, Health, Subscriptions, Other.
- Multi-select chips (toggle on/off). Default: all selected.
- Selected chip: `--surface-elevated-dark` bg + yellow border. Unselected: `--surface-card-dark` bg + hairline border.
- Help text: `"Selected categories start with £0 planned. Adjust limits after."` (currency symbol from step 2).
- At least 1 category required. Inline error: `"Select at least one category."`.
- On "Next": POST category rows + upsert `onboarding_progress(step=4)`.

**Step 5 — Review & Create:**

- Summary card: budget name, currency, type, N categories selected.
- No editable fields — all read-only with "Back" to revisit.
- Primary CTA: `"Create budget"` (yellow filled button).
- On submit: PATCH `onboarding_progress(step=5, completed_at=now)` → redirect to `/budgets/[id]`.

**Action row (all steps):**

- Left: `"Back"` ghost button (neutral — NOT yellow). Hidden on step 1.
- Right: `"Next"` primary button (yellow filled) on steps 1–4. `"Create budget"` on step 5.
- Step 2–4: `"Skip"` ghost button (neutral) between Back and Next — optional steps only (not step 1 name, not step 5 review).
- Skip moves to the next step without saving that step's data; budget row is unaffected.

**Resume (D-08):** Layout/middleware guard reads `onboarding_progress`. If `completed_at IS NULL` → force-redirect to `/budgets/new` at `?step=[saved_step]`. Wizard reads `?step` param on mount to restore position.

**Stray `/onboarding` route:** Redirect 301 to `/budgets/new`. Do not hard-delete (may be linked externally).

### Share-Link Recipient Join Page — `/budgets/join/[token]`

**Route type:** Public for the view (no auth gate). Auth required to accept (Better Auth session check before POST).

**States:**

| State                       | Content                                                                                                                |
| --------------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| Valid link, unauthenticated | `<JoinPageCard>` with budget name, inviter info, "Accept invitation" CTA → redirects to sign-in, then back to this URL |
| Valid link, authenticated   | Budget name + "Join [budget name]" CTA (yellow filled)                                                                 |
| Expired / Revoked           | Error card: `"This invitation has expired."` + `"Ask the budget owner to send a new link."`                            |
| Already used                | Info card: `"You've already joined this budget."` + `"Go to [budget name]"` link                                       |
| Unknown token (404)         | Error card: `"Invitation not found."` + `"Return home"` link                                                           |
| Accepting (loading)         | CTA button shows spinner, text `"Joining…"`, disabled                                                                  |

**Layout:** Full-viewport centered, max-width 400px. Card on `--surface-card-dark`. No nav chrome (public page, no sidebar or top nav).

**Join card anatomy:**

- App logo/wordmark at top (32px height).
- Budget name: Heading role (16px/600).
- Invitation copy: Body role (14px/400, `--muted`).
- CTA button: Full-width yellow `"Join budget"` (authenticated) or `"Sign in to accept"` (unauthenticated, yellow text link styled as button).
- Expiry caption below CTA: `"This link expires {relative date}"` in muted caption (12px/500).
