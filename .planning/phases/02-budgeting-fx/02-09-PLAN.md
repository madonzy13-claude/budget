<objective>
Wrap Phase 2 with the read-side capabilities (search/filter/bulk re-categorize) and projection durability (reconciliation cron + replay CLI). Search uses the GIN index on `note_tsv` (created in plan 02-06) plus B-tree indexes on `(tenant_id, transaction_date)`, `(tenant_id, category_id, transaction_date)`, `(tenant_id, account_id, transaction_date)`. Cursor pagination via `(transaction_date, id)`. Bulk re-categorize loops `insertCorrection` (plan 02-07) in a single `withTenantTx` for atomicity. Reconciliation cron runs hourly and compares projection rows against fresh aggregate; auto-repairs small drift, logs + alerts on large drift. CLI `bun run replay:budgeting` rebuilds projections from ledger for an operator-supplied date range.

Purpose: This plan finalizes EXPN-09, EXPN-10, ENGR-14 — the last three Phase-2 requirements that don't fit cleanly into the write-side plans. Without it, the user has no way to find old transactions and no operator recovery path if projections drift.

Output: search/filter UI per UI-SPEC § Search/Filter; bulk action bar per UI-SPEC § Multi-select for bulk re-categorize; reconciliation cron registered hourly; replay CLI script; e2e for search-filter happy path + bulk-recategorize.
</objective>

    1. **reconcile-projections.ts** — implement per behavior. SQL:
       ```sql
       WITH latest AS (
         SELECT * FROM budgeting.expense_ledger
         WHERE tenant_id = $1 AND kind = 'EXPENSE'
           AND id NOT IN (SELECT corrects_id FROM budgeting.expense_ledger WHERE corrects_id IS NOT NULL AND tenant_id = $1)
           AND transaction_date BETWEEN $2 AND $3
       ),
       fresh AS (
         SELECT category_id, date_trunc('month', transaction_date)::date AS month_start_date,
                SUM(amount_default) AS normal_amount, currency_default AS currency
         FROM latest WHERE category_id IS NOT NULL
         GROUP BY category_id, date_trunc('month', transaction_date)::date, currency_default
       )
       SELECT
         f.category_id, f.month_start_date, f.normal_amount AS fresh_normal,
         COALESCE(p.normal_amount, '0') AS proj_normal,
         f.currency
       FROM fresh f
       LEFT JOIN budgeting.spending_by_category_month p
         ON p.tenant_id = $1 AND p.category_id = f.category_id AND p.month_start_date = f.month_start_date
       WHERE ABS(COALESCE(p.normal_amount, '0')::numeric - f.normal_amount::numeric) > 0.005;
       ```
