- Touch targets: minimum 44×44px effective area for all tappable rows (row tap-area, not visible padding)

- Pending drafts badge: 20×20px pill with 4px horizontal overhang on nav icon

## Typography

All type from DESIGN.md tokens. BinancePlex for every monetary/numeric value; BinanceNova for all copy.

Phase 2 uses 4 sizes and 2 weights. Page titles use the same 16px/600 token as section titles — Phase 2 screen headings are compact enough that a 24px tier adds no hierarchy value over the card/drawer title context they appear in.

| Role                   | Token            | Size | Weight | Line Height | Font        | Use                                                                                                                    |
| ---------------------- | ---------------- | ---- | ------ | ----------- | ----------- | ---------------------------------------------------------------------------------------------------------------------- |
| Title (page + section) | `title-sm`       | 16px | 600    | 1.4         | BinanceNova | Screen headings, card headers, drawer titles, group labels (Accounts, Categories, Transactions, Edit history)          |
| Body / button          | `body-md`        | 14px | 400    | 1.5         | BinanceNova | Row labels, descriptions, form hints, empty state body; buttons use 14px/600 (same size, heavier weight — see note)    |
| Caption / meta         | `caption`        | 12px | 400    | 1.4         | BinanceNova | FX freshness badge, "edited" badge, share-override badge, secondary metadata, cadence badges, filter result counts     |
| Amount display         | `number-display` | 40px | 600    | 1.1         | BinancePlex | Transaction capture amount input, FX preview converted amount — prominent numeric entry point (DESIGN.md `display-md`) |

**Button weight note:** CTA labels use `body-md` size (14px) at weight 600. This is not a separate size — it is the same 14px token rendered at the heavier of the two declared weights.

**Number tokens (use body-md size, weight 600, BinancePlex font):**

- `number-md` — 16px/600/BinancePlex: transaction list amounts, account balance, category limit values (uses title-sm size)
- `number-sm` — 14px/600/BinancePlex: budget bar spent/limit sub-labels, percentage shares, FX rate inline (uses body-md size)

Weights in use: **400** (body copy, captions, metadata) · **600** (titles, buttons, all numeric values, amount display).

**Accent (`primary` #FCD535) reserved for:**

- "Save" / "Confirm transaction" primary CTA button background
- "Add Transaction" floating action button background
- "Apply Template" button background
- Active nav item indicator dot or underline
- Budget bar fill when spending is within normal limit

**`trading-up` (#0ecb81) reserved for:**

- Income transaction amount text
- Budget bar fill color when at 0–80% of limit (good state)
- Positive account balance text

**`trading-down` (#f6465d) reserved for:**

- Expense transaction amount text (not all expenses — only when visually flagging the debit)
- Budget bar fill when spending exceeds cushion limit (over budget)
- Delete/destructive action button text and border
- Error messages and validation failures

**Budget bar fill states (three-state):**

- 0–80% spent: `trading-up` green fill
- 81–100% spent: `primary` yellow fill (warning, approaching limit)
- > 100% spent: `trading-down` red fill (over budget)

## Component Inventory

All components built from existing primitives in `apps/web/src/components/ui/`.

### Transaction Capture Form

**Surface:** `sheet` (right-side drawer, 480px wide on desktop; full-screen on mobile <768px)
**Background:** `surface-card-dark`
**Fields:**

- Amount input: `input` component, `number-display` typography (40px/600/BinancePlex), placeholder "0.00"
- Currency picker: `select` component, inline after amount — shows ISO code + flag emoji
- Date: `input` type=date, styled to dark canvas
- Kind: `tabs` component with 3 tabs — Expense / Income / Transfer
- Category: `select` with grouped options (group = category group label in `muted` color)
- Account: `select`
- Note: `input` type=text, optional
- FX preview (when currency ≠ workspace default): inline row below amount — "≈ {amount} {currency} @ {rate}, {provider}, {freshness}" using `caption` + `number-sm` tokens; `info` color icon if stale
- Idempotency-Key: generated client-side UUID on drawer open; submitted as HTTP header — no UI element

**Buttons:**

- Primary: "Save expense" / "Save income" / "Save transfer" — `button-primary` (yellow)
- Cancel: `button-tertiary-text`

**Transfer special:**

- Two account pickers (From / To)
- No category picker
- FX preview shown between accounts if currencies differ

### Transaction List (Ledger View)

**Surface:** Full-width list within `surface-card-dark` card container; `rounded-xl`
**Row anatomy** (12px vertical padding, `hairline-on-dark` divider):

- Left: date in `caption`/`muted`, category icon + name in `body-md`
- Center: note in `body-md`/`muted` (truncated to 1 line)
- Right: amount in `number-md`/BinancePlex — green (`trading-up`) for income, red (`trading-down`) for expense; transfer shows neutral `body`
- Below amount (right): FX freshness badge if `fx_rate_date < transaction_date` — `caption` text "rate {humanized-age}" in `muted-strong` on `surface-elevated-dark` pill
- Below category (left): "edited" badge if row has been corrected — `caption` "edited" in `muted-strong` on `surface-elevated-dark` pill; clicking opens history side panel

**Correction history side panel:**

- `sheet` left-side, 360px
- Title: "Edit history" in `title-sm`
- Each version row: actor name, timestamp, changed fields — `body-md`; original row styled in `muted`

**Multi-select for bulk re-categorize:**

- `checkbox` appears on row hover (desktop) or via long-press (mobile)
- Bulk action bar slides up from bottom: "Re-categorize {N} transactions" → `select` picker → "Apply" button

### Accounts List

**Surface:** `card` container, `surface-card-dark`, `rounded-xl`, 24px padding
**Sections:** Assets (cash, checking, savings, investment) / Liabilities (credit card, loan) — `title-sm` group label
**Account row:** account name `body-md` · currency · balance in `number-md`/BinancePlex (asset: `on-dark`; liability: `trading-down`)
**Actions:** Edit icon (pencil), Archive icon (archive-box) — appear on row hover; `muted` color, 20px icon

**Icon-only action accessibility contract:**

- Edit button: `aria-label="Edit {account name}"` — executor must not omit
- Archive button: `aria-label="Archive {account name}"` — executor must not omit
- Both buttons: `role="button"`, minimum 44×44px touch target

**"Adjust balance" action:** opens inline row with amount input + "Save adjustment" — records BALANCE_ADJUSTMENT ledger row
**Add button:** "+ Add account" — `button-secondary-on-dark` at bottom of card

**Account detail / edit:**

- `dialog` component, 480px
- Fields: name, account type (`select`), currency (`select`, locked after creation with tooltip), opening balance
- Archive: `button-tertiary-text` in `trading-down` color, at bottom of dialog

### Categories List

**Surface:** `card` container, grouped by category group
**Category row:** icon + name `body-md` · normal limit `number-sm`/BinancePlex · cushion limit `number-sm`/BinancePlex (muted if same as normal) · budget bar (see above) · "override" badge if contribution shares differ from global
**Expand row:** clicking row opens inline contribution shares editor (see below)
**Edit icon:** opens edit `dialog`; `aria-label="Edit {category name}"` required

**Category edit dialog (480px):**

- Name, group (text input with datalist), icon picker (emoji)
- Normal limit: amount + currency inputs
- Cushion limit: amount + currency inputs (pre-filled = normal)
- Contribution shares section (SHARED workspaces only):
  - Toggle "Override for this category" — `checkbox`
  - When toggled on: per-member `input` type=number (percentage), with live sum display "Currently {N}% — must equal 100%"
  - Save disabled + sum counter shown in `trading-down` if sum ≠ 100%
  - Sum counter shown in `trading-up` when sum = 100%

**Budget mode toggle (workspace settings):**

- `tabs` component: Normal / Cushion
- Persisted as `workspaces.budget_mode`; affects which limit column all budget bars read from

### Budget Templates

**Surface:** Separate screen, `card` list
**Template row:** name `body-md` · category count `caption`/`muted` · "Apply" button (`button-secondary-on-dark`) · Edit / Delete icons
**Apply flow:** confirms target month via `dialog` → "Apply to {Month YYYY}" `button-primary`
**Create/edit template:** `dialog` with name field + category-limit rows (add/remove categories, set normal + cushion amounts)

### Pending Recurring Drafts Inbox

**Nav badge:** numeric count badge on nav item — `primary` yellow background, `on-primary` text, 20px pill; shows count of pending + overdue drafts
**Surface:** Dedicated screen "Pending transactions" or drawer triggered from nav badge
**Draft row anatomy:**

- Rule name + amount + category — `body-md`
- Due date — `caption`/`muted`; if overdue, date shown in `trading-down`
- Three action buttons inline: "Confirm transaction" (`button-primary` small 28px), "Edit & confirm" (`button-secondary-on-dark` small), "Skip this period" (`button-tertiary-text`)
- Skip triggers `alert-dialog`: "Skip {rule name} for {period}? This period will not be recorded." — Confirm / Cancel

**Edit-and-confirm flow:**

- Clicking "Edit & confirm" opens the transaction capture `sheet` pre-filled with draft values
- On save from sheet: inserts ledger row and deletes draft

### Recurring Rules Manager

**Surface:** `card` list within Settings or dedicated Recurring section
**Rule row:** name, cadence badge (`caption` — "Monthly" / "Weekly · Mon"), next due `caption`/`muted`, amount `number-sm`
**Create / edit rule `dialog` (480px):**

- Name, amount, currency, category, account, note
- Cadence: `select` — Monthly / Weekly
- If weekly: day-of-week `select` (Monday–Sunday)
- Anchor day: numeric input (for monthly: day of month 1–28)
- Edit mode shows: "Apply changes to: This period only [default] / This and all future periods [checkbox]"

### Search & Filter

**Surface:** filter bar above transaction list
**Search:** `input` type=search, `search-input-on-dark` style (background `surface-card-dark`, rounded `rounded-lg`)
**Filter chips:** date range · category · account · kind (Expense/Income/Transfer) — each a `badge` variant (ghost when inactive, filled `surface-elevated-dark` when active)
**Filter panel:** `popover` triggered by filter icon; contains date range pickers, category multi-select, account multi-select, kind toggle
**Results count:** `caption`/`muted` — "{N} transactions" shown when filters active

### FX Freshness Badge

**Spec (from D-03-a, user-given requirement):**

- Always visible on any transaction row or FX preview where `fx_rate_date < transaction_date`
- Text: humanized age — "rate from Friday", "rate 2h 15m old", "rate 3 days old"
- Style: `caption` (12px/400) text in `muted-strong`, pill background `surface-elevated-dark`, 4px border-radius, 4px vertical padding × 8px horizontal
- Icon: `info` circle, 12px, `info` blue color (#3b82f6)
- No click action, no dismiss — purely informational

**FX preview in deposit form (D-02-d):**

- Full row: "1000 PLN ≈ 232.50 EUR @ 0.2325, Frankfurter, 2026-05-09"
- Amount in `number-sm`/BinancePlex; label text in `caption`/`muted`
- Stale indicator: if rate >60 min old at save time, server returns 409 and client shows `alert-dialog`: "Rate has changed. New rate: {rate}. Save with updated rate?" — Confirm / Cancel

### Shares Mismatch Banner (D-02-c)

**Trigger:** member added or removed from SHARED workspace
**Surface:** sticky banner below top nav, full-width
**Background:** `surface-elevated-dark`; left border 4px `trading-down`
**Copy:** "Contribution shares need updating before new transactions can be saved. [Update shares →]" — link in `primary` yellow
**Dismissal:** only dismissed when owner completes share redistribution (sum = 100%)
**New transaction save:** blocked — Save button shows `tooltip` "Update workspace shares first"

### Workspace Settings (budget-related additions)

**Surface:** existing settings screen, new "Budget" tab
**Timezone display:** read-only field showing current IANA timezone + offset; "Edit" link opens confirmation `dialog` (warns: "Future periods will use new timezone. Historical aggregates are not recalculated.")
**Budget mode:** `tabs` — Normal / Cushion
**Global contribution shares (SHARED only):** per-member percentage inputs with live sum counter (same pattern as category override)

## Interaction Contracts

### Idempotency-Key Client Behavior

- Generate UUID v4 on every mutation form open (drawer/dialog open event)
- Store in component state; do not regenerate on re-render
- Submit as `Idempotency-Key` HTTP header on every mutating request
- On 409 "key used with different body": show error copy (see above)
- On network timeout + retry: reuse same key — server returns cached response

### FX Preview Live Update

- Fetch rate on form mount if `currency ≠ workspace.default_currency`
- Re-fetch on currency-change event (debounced 400ms)
- Display preview row immediately after fetch; show skeleton while loading
- Rate held in form state; submitted with save request (not re-fetched at save time)
- If held rate age >60 min at server validation: server returns 409 with fresh rate; client shows `alert-dialog` (see above)

### Edit-via-Correction UX

- "Edit" button on transaction row opens transaction capture sheet pre-filled with latest values
- Sheet title: "Edit transaction" (not "New transaction")
- On save: inserts correction row (`corrects_id = original.id`); original row never mutated
- Optimistic UI: immediately shows updated values in list; reverts on error
- "Edited" badge appears on row after save; badge click opens correction history sheet

### Pending Drafts Confirmation Flow

- Draft row shows three inline buttons (Confirm transaction / Edit & confirm / Skip this period)
- Confirm transaction: immediate POST to confirm endpoint; row animates out; badge count decrements
- Edit & confirm: opens transaction capture sheet pre-filled; saving from sheet confirms draft
- Skip this period: opens `alert-dialog`; on confirm, row animates out; badge count decrements

### Budget Bar Update

- Budget bars update optimistically on transaction save
- Re-fetches projection from server on: page/drawer open, after any transaction mutation
- Three fill states: green (0–80%), yellow (81–100%), red (>100%) — smooth CSS transition on value change

### Workspace Shares Sum Enforcement

- Live percentage sum computed on every input change
- Sum display updates immediately below inputs
- Save button disabled when sum ≠ 100% (within 0.001% tolerance)
- Counter text color: `trading-down` when < or > 100%; `trading-up` when = 100%
