### 2. Rewrite i18n JSON files — codemod + manual review

- **Files:**
  - `apps/web/messages/en.json`
  - `apps/web/messages/pl.json`
  - `apps/web/messages/uk.json`
- **Action:** Per PATTERNS §9 (subtree map). Codemod via `jq`, then manual review per file.
  - **Step A — jq codemod (run per file):**
    ```bash
    for f in en pl uk; do
      jq '
        # Rename top-level workspaces -> budgets, preserve subtree
        if has("workspaces") then .budgets = .workspaces | del(.workspaces) else . end
        # Rename top-level singular workspace -> budget
        | if has("workspace") then .budget = .workspace | del(.workspace) else . end
        # Rename nav.workspaces -> nav.budgets
        | if .nav and (.nav | has("workspaces")) then .nav.budgets = .nav.workspaces | del(.nav.workspaces) else . end
        # Rename nav.accounts -> nav.wallets
        | if .nav and (.nav | has("accounts")) then .nav.wallets = .nav.accounts | del(.nav.accounts) else . end
        # Rename budgeting.accounts -> budgeting.wallets, preserve subtree
        | if .budgeting and (.budgeting | has("accounts")) then .budgeting.wallets = .budgeting.accounts | del(.budgeting.accounts) else . end
        # Drop budgeting.wallets.form.scopeLabel (D-13)
        | if .budgeting and .budgeting.wallets and .budgeting.wallets.form then del(.budgeting.wallets.form.scopeLabel) else . end
      ' "apps/web/messages/$f.json" > "apps/web/messages/$f.json.tmp" && mv "apps/web/messages/$f.json.tmp" "apps/web/messages/$f.json"
    done
    ```
  - **Step B — manual review of each file** (codemod does NOT translate the English-language strings inside; that's a per-locale judgment call):
    - **EN (en.json):** Edit every English string that mentions "Workspace(s)" → "Budget(s)", "workspace" → "budget" (lowercase same), "Account(s)" → "Wallet(s)" inside `budgets.*`, `budget.*`, `budgeting.wallets.*`. Examples from PATTERNS §9 lines 749-777: `"heading": "Create your first workspace"` → `"heading": "Create your first budget"`; `"cta": "Create workspace"` → `"cta": "Create budget"`; `"label": "Workspace name"` → `"label": "Budget name"`; `"placeholder": "e.g. Family budget"` stays; `"private": "Private -- just me"` stays. PATTERNS §9 lines 798-810: `"title": "Accounts"` → `"title": "Wallets"`, `"addButton": "Add account"` → `"addButton": "Add wallet"`, etc.
    - **PL (pl.json):** Apply the analogous transformation in Polish — likely `"Przestrzenie robocze"`/`"Workspace"` → `"Budżety"`/`"Budżet"`; `"Konto"`/`"Konta"` (account/accounts) → `"Portfel"`/`"Portfele"` (wallet/wallets). Translator-quality check is the user's responsibility; executor uses these mappings as scaffolding and adds `// TODO: native-PL review` comments via a sidecar `.md` if confidence is low.
    - **UK (uk.json):** Same in Ukrainian — `"Робочий простір"` (workspace) → `"Бюджет"`, `"Рахунок"` (account) → `"Гаманець"` (wallet). Same review caveat.
    - **`nav.workspaces`** display string in EN was `"Workspaces"` (PATTERNS line 785) → `"Budgets"`. `nav.accounts` `"Accounts"` (line 790) → `"Wallets"`.
  - **Step C — additional leaf strings in component code** (PATTERNS §9 final bullet):
    - `grep -rn "i18n.*workspaces\\.\\|i18n.*accounts\\." apps/web/src/components apps/web/src/app` — for each hit, update the i18n key path. Examples:
      - `useTranslations('workspaces')` → `useTranslations('budgets')`
      - `t('workspaces.empty.heading')` → `t('budgets.empty.heading')`
      - `t('nav.accounts')` → `t('nav.wallets')`
      - `t('budgeting.accounts.title')` → `t('budgeting.wallets.title')`
  - **Drop `scope` references in i18n** — beyond `budgeting.wallets.form.scopeLabel`, grep for any `scope` translation keys in `budgeting.*` subtrees and remove (D-13).
- **Verify:**
  - `bun --cwd apps/web run test apps/web/test/i18n/v11-key-rename.test.ts` — GREEN.
  - `jq 'has("workspaces") or has("workspace")' apps/web/messages/en.json` returns `false`. Same for pl + uk.
