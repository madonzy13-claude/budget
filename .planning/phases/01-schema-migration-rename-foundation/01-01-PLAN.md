# Plan 01-01 — Schema Migration, Post-Migration Lockstep & CI Gate Retarget

## Goal

End state: `drizzle/0012_phase01_v11_rename.sql` exists and, when applied to a freshly-nuked dev DB followed by `apps/migrator/post-migration.sql`, produces a Postgres schema where:

- `tenancy.budgets` / `tenancy.budget_members` / `tenancy.shared_budget_member_shares` / `tenancy.budget_invitations` replace the `workspace*` counterparts.
- `budgeting.wallets` replaces `budgeting.accounts` (with new `wallet_type` ENUM column SPENDINGS/CUSHION/RESERVE, and `scope` dropped).
- `budgeting.budget_mode_history` replaces `budgeting.workspace_budget_mode_history` with `workspace_id` → `budget_id`.
- `budgeting.workspace_share_dirty` → `budgeting.budget_share_dirty` (FK col rename too).
- `budgeting.balance_adjustments.workspace_id` → `budget_id`, `account_id` → `wallet_id` (table retained per D-12).
- Legacy columns dropped wherever present: `budgeting.expense_ledger.kind`, `budgeting.expense_ledger.account_id`, `budgeting.expense_ledger.to_account_id`, `budgeting.expense_ledger.direction`, `budgeting.accounts.scope` (consumed by rename), `budgeting.categories.scope` (D-13). Silently no-op the ones that don't currently exist (RESEARCH §Q on MIG-03 ledger).
- New columns added: `budgeting.categories.sort_index INTEGER NOT NULL DEFAULT 0`, `tenancy.budgets.cushion_mode_enabled BOOLEAN NOT NULL DEFAULT false`, `budgeting.category_limits.cushion_amount_cents BIGINT` (parallel SCD-2 column; D-11 keeps existing `cushion_amount` untouched — MIG-05 wording supplements it).
- New table `budgeting.tasks (id, tenant_id, budget_id, kind text+CHECK, payload_json jsonb, status text+CHECK, created_at, resolved_at)` with `FORCE ROW LEVEL SECURITY`, `tasks_tenant_isolation` policy, FK `budget_id` → `tenancy.budgets(id) ON DELETE CASCADE`, index `(budget_id, status)`.
- `identity.accounts` (Better Auth provider table) is **untouched** (RESEARCH §Q1).
- `make ci-gate` is 6/6 green — 5 backend `tests/tenant-leak/*.test.ts` plus `apps/web/e2e/cross-tenant-cache.spec.ts`.

Dev DB is wiped and reseeded as recovery path (MIG-09); no data migration required.
