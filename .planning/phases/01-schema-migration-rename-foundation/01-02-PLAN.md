# Plan 01-02 — Domain & Application Layer Rename + Dropped-Column Cleanup

## Goal

End state: `bun run typecheck` from repo root is green (zero TS errors). `bunx dependency-cruiser --config .dependency-cruiser.cjs packages apps` reports no `domain → drizzle-orm` or `domain → adapters` import violations. Every domain class, application service, repo, port, and worker handler that previously referenced `Workspace`/`Account` now references `Budget`/`Wallet`. Every reference to a column dropped by Plan 01-01 (`transactions.kind`, `transactions.account_id`, `to_account_id`, `direction`, `accounts.scope`, `categories.scope`) is gone from packages/_ and apps/worker/_. The Better Auth `organizationId` JS field name is preserved on the membership repo per the CONTEXT addendum, even though the DB column is now `budget_id`.

Phase 1 keeps route bodies in their v1.0 shape (D-07); this plan does ONLY the minimum compile-fix on application/domain — Plan 01-03 owns the Hono route layer.
