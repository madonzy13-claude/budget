threat_model:

    stride: Spoofing (account takeover via email change) / Information Disclosure
    surface: authClient.changeEmail — changing the login identifier
    mitigation: "Better Auth requires an authenticated session to call changeEmail and (current email verified) sends a confirmation link to the OLD address before the change commits, so a hijacked-but-unconfirmed session cannot silently move the account. The new address must be re-verified (email_verified=false until clicked). email_hash is recomputed server-side in a trusted databaseHooks after-hook (not client-trusted)."
  - id: T-10-04
    stride: Tampering (PII-lookup integrity)
    surface: encrypted email columns vs Better Auth plain-email write
    mitigation: "changeEmail writes only the plain email column; the after-hook recomputes email_hash via keyStore.emailHash so the UNIQUE index + findByEmail stay correct. Without the hook, login-by-email silently breaks — the integration test (email-change-hash.test.ts) is the guard."

BETTER AUTH WIRING (RESEARCH §1, §1a — exact):

```ts
user: {
  // ...existing additionalFields (locale, displayCurrency) ...
  changeEmail: {
    enabled: true,
    sendChangeEmailConfirmation: async ({ user, newEmail, url }) => {
      await opts.emailSender.send({
        to: user.email,                 // OLD address (confirm link)
        template: "change-email",
        vars: { url, newEmail },
        locale: pickLocale((user as { locale?: string }).locale),
      });
    },
    updateEmailWithoutVerification: false,
  },
},
databaseHooks: {
  user: {
    create: { after: async (user) => { /* EXISTING — do not change */ } },
    update: {
      after: async (user) => {
        const hash = await opts.keyStore.emailHash(user.email as string);
        await withUserContext(UserId(user.id as string), async (tx) => {
          await tx.execute(sql`UPDATE identity.users SET email_hash = ${Buffer.from(hash)} WHERE id = ${user.id}::uuid`);
        });
      },
    },
  },
},
```
