<objective>
Build the **Danger Zone** account-deletion flow (GDPR right-to-delete). Enable Better Auth `user.deleteUser` with a
`beforeDelete` hook that runs the **application-level cascade** (RESEARCH §2 — tenancy/shared_kernel have NO DB FK to
identity, so nothing cascades automatically). Deletion purges the user's keys, memberships, invitations, and any
**solely-owned** budget (PRIVATE or member-less SHARED) with all its tenant data; it **blocks** when the user solely
owns a SHARED budget that still has other members. The UI gates on a typed `DELETE` confirmation + password and signs
the user out. TDD: the cascade is proven by a real-Postgres integration test first.

Purpose: CONTEXT decision 6 + RESEARCH §2. autonomous:false — the cascade is irreversible; a human confirms scope
(and the SHARED-block decision) during execution. Mounts into the User-pill Danger Zone slot from plan 10-02.
</objective>

```ts
user: {
  // ...changeEmail (from 10-03), additionalFields...
  deleteUser: {
    enabled: true,
    beforeDelete: async (user) => {
      const uid = user.id as string;
      await withUserContext(UserId(uid), async (tx) => {
        // 1. Block if sole owner of a SHARED budget that still has other members.
        const blocked = await tx.execute(sql`
          SELECT b.id FROM tenancy.budgets b
          WHERE b.owner_user_id = ${uid}::uuid AND b.kind = 'SHARED'
            AND (SELECT count(*) FROM tenancy.budget_members m WHERE m.budget_id = b.id) > 1
          LIMIT 1`);
        if (blocked.rows?.length) {
          throw new APIError("BAD_REQUEST", { message: "Transfer ownership or remove other members before deleting your account." });
        }
        // 2. Cascade-delete solely-owned budgets + their tenant data (all schemas keyed by tenant_id = budget.id).
        //    Enumerate owned budget ids, then delete budgeting/reserves/cushion/investments/shares rows for those
        //    tenant_ids, then the budget_members + budgets rows. (List the exact per-schema tables to purge — derive
        //    from the schema files in read_first; do NOT guess table names.)
        // 3. Delete this user's own personal rows: budget_members (user_id), budget_invitations (inviter_id),
        //    shared_kernel.user_keys (user_id), identity.verifications for the user.
      });
    },
  },
},
```
