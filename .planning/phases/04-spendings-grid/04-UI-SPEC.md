## Spacing Scale

Phase 4 uses the project's existing **DESIGN.md 4-multiple scale** verbatim from `global.css`. **No new tokens.**

| Token             | Value | Phase 4 usage                                                                                                                                                                               |
| ----------------- | ----- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `spacing.xxs`     | 4px   | Inline icon-to-label gaps inside floating action chips; row-internal vertical compact spacing; drag-grip hit-area outer ring                                                                |
| `spacing.xs`      | 8px   | Vertical gap between txn rows inside a column; column header internal row-to-row gap; quick-entry input internal padding; floating action chip outer padding; slider field-row internal gap |
| `spacing.sm`      | 12px  | Column internal horizontal padding (left + right edges of each column); slider field cluster vertical gap; month-navigator button-to-label gap                                              |
| `spacing.md`      | 16px  | Slider section padding; slider footer button-row gap; column-header total stack vertical padding; mobile column width adjustments; transition-row internal padding                          |
| `spacing.lg`      | 24px  | Slider outer body padding (matches Phase 3 card precedent); month-navigator-to-grid vertical gap; sticky-tabs-to-grid top gap; gap between column groups when grouped (future)              |
| `spacing.xl`      | 32px  | Grid outer horizontal page gutter on desktop (matches Phase 3 top-nav gutter); grid-to-page-bottom safety padding                                                                           |
| `spacing.xxl`     | 48px  | **Not used in Phase 4.** Reserved for hero/onboarding spacing — grid is dense product surface (DESIGN.md L450).                                                                             |
| `spacing.section` | 80px  | **Not used in Phase 4.** Same reason as above.                                                                                                                                              |

**Phase 4 fixed dimensions (DESIGN.md derived, no new tokens):**

| Element                               | Dimension                            | Source                                                                                                          |

| Category column width (mobile)        | **140px** fixed                      | Lower end of the same range for horizontal-scroll density                                                       |

## Typography

Phase 4 uses **3 type roles + 1 numeric role** drawn from DESIGN.md type ramp and pre-defined utilities in `global.css`. No new sizes invented. **Mirrors Phase 3 exactly — keeps the BDP unified.**

| Role    | Token                         | Size | Weight | Line Height | Font stack                                                             | Phase 4 usage                                                                                                                                                                |
| ------- | ----------------------------- | ---- | ------ | ----------- | ---------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Body    | `body-md`                     | 14px | 400    | 1.5         | Inter (`--font-sans`)                                                  | Transaction note text, draft row body, slider field labels, slider helper text, empty-column state text, error message body                                                  |
| Label   | `caption` (`.text-caption`)   | 12px | 500    | 1.4         | Inter                                                                  | Column header row labels ("planned" / "overspent" / "reserves used" / "balance" sub-line captions), slider field labels, month-nav caption                                   |
| Title   | `title-sm` (`.text-title-sm`) | 16px | 600    | 1.4         | Inter                                                                  | Category name in column header (row 1); slider header title ("New transaction" / "Edit category"); month label ("May 2026")                                                  |
| Numeric | `num-md` (`.text-num-md`)     | 16px | 500    | 1.4         | IBM Plex Sans (`--font-numeric`), `font-variant-numeric: tabular-nums` | All currency values: planned/cushion (row 2), overspent (row 3), reserves-used (row 4), balance (row 5), txn amounts, draft amounts, quick-entry placeholder + entered value |

**Numeric role secondary size** for inline FX-pair (slider only):

| Role            | Token                     | Size | Weight | Line Height | Font stack    | Usage                                                                        |
| --------------- | ------------------------- | ---- | ------ | ----------- | ------------- | ---------------------------------------------------------------------------- |
| Numeric (small) | `num-sm` (`.text-num-sm`) | 14px | 500    | 1.4         | IBM Plex Sans | Slider FX-display row: `"5.00 USD · ~4.20 EUR @ 0.84 (2026-05-11)"` (TXN-06) |

**Weights declared: exactly 2 — 400 (body) and 600 (title).** Numeric role uses weight 500 per DESIGN.md `number-md` token — a single tabular-numerals exception that is not a new editorial weight (ships in `global.css` already, identical to Phase 3 carry-forward).

**Body line-height:** 1.5 (DESIGN.md `body-md`). **Title line-height:** 1.4 (DESIGN.md `title-sm`). **Caption line-height:** 1.4.

**Don't:**

- Don't render any currency value in Inter — every monetary number routes through `.text-num-md` or `.text-num-sm` (IBM Plex Sans + tabular-nums) per DESIGN.md L594.
- Don't introduce `display-*` sizes — grid is dense product surface, not marketing.
- Don't soften title weight to 400 — DESIGN.md L604.
- Don't bold txn notes, draft titles, or quick-entry placeholders — body weight 400 only.

**Accent (10% yellow) reserved-for list — Phase 4 EXACT inventory. No other element may be yellow.**

1. **`Confirm` button on `DraftRow`** (revealed via single-click on draft row). `button-primary-pill` style: background `--primary` + text `--on-primary`. The yellow primary action that promotes the draft unchanged (RECR-04).
2. **Dashed left-border decoration** on `DraftRow` — **3px dashed `--primary` left border**. **Decorative only** per D-PH4-R1; the solid yellow `Confirm` button still carries the "yellow = primary action" rule. The dashed border is the visual signal that "this row needs your attention" without claiming primary-action voltage.
3. **Sticky pill tab "Spendings" — yellow active background** (Phase 3 carry-forward; not redefined here, but it remains the only yellow on the BDP frame above the grid).
4. **Yellow ring around currently-focused column header** when month nav is keyboard-driven — **2px `--primary` ring at +2px offset**. Subtle wayfinding cue distinct from `--info-ring` blue (used for input focus). Appears only on column-level focus transitions, never on idle.

**Destructive (red) reserved-for list — Phase 4 EXACT inventory:**

1. **Trash icon stroke** in `TransactionRow` revealed actions — stroke `--destructive` resting. (Confirmation dialog before delete is fired.)
2. **Delete button inside `TransactionSlider` footer** — `Button variant="destructive"` from `ui/button.tsx`. Appears in edit mode only.
3. **Header row 3 (overspent)** — text color `--destructive` when `overspent > 0`. Text-color only, no fill (DESIGN.md L603).
4. **Header row 5 (balance)** — text color `--destructive` when `balance < 0`. (Mirrors row-3 rule.)
5. **Quick-entry input red ring** — 2px `--destructive` ring when an optimistic POST has failed and the most recent unsent row is visible. Ring clears when user retries successfully OR taps the inline retry icon. (Distinct from `--info-ring` blue used for normal focus.)

**Recurring-draft row exact treatment (D-PH4-R1):**

- Background: `--surface-elevated-dark` (`#2b3139`).
- Left border: **3px dashed `--primary` (`#fcd535`)**.
- Border-left treatment: `border-left: 3px dashed var(--primary)` — pure CSS, no SVG.
- Text: same as standard txn row (`--body-on-dark`).
- Amount: `.text-num-md`.
- Hover: **no change** (hover does nothing — D-PH4-INT1).
- Single-click revealed actions: chip background `--surface-elevated-dark`, lifted +2px shadow-less; chip contains `[Confirm yellow pill][Pen ghost][Dismiss ghost]`.

## Component Interaction Contracts

Universal interaction model — locked by CONTEXT D-PH4-INT1…6 and reiterated here as the highest-precedence rule for every component below.

| Primitive               | Trigger                               | Behavior                                                                                                                                                         |
| ----------------------- | ------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Single click**        | Any interactive row or cell           | Reveals floating action chips inline next to the row. Reveals once; outside-click or `Escape` collapses. **Hover NEVER reveals anything.**                       |
| **Double click**        | Cell containing visible-only fields   | Cell enters inline edit mode. Cell becomes an input. `Enter` saves, `Escape` cancels. **Scope:** txn rows = amount only; category cells = NEVER (always slider). |
| **Long press**          | Mobile equivalent of single click     | 300ms long-press on touch devices = single-click equivalent. Reveals action chips identically.                                                                   |
| **Pen icon click**      | Pen icon inside revealed action chips | Opens the corresponding slider (`TransactionSlider` or `CategorySlider`) pre-filled. **Slider is the ONLY surface for full-field edit.**                         |
| **Pointerdown on grip** | Drag grip icon on column header       | Initiates dnd-kit drag. Grip is the only drag-start surface (D-PH4-D3). Body of column never drags.                                                              |
| **Outside click**       | Anywhere outside revealed actions     | Collapses revealed action chips. If a cell is in inline-edit mode, commits the edit (Enter-equivalent) per D-PH4-INT2 explicit-lock-rule.                        |
| **Escape**              | Anywhere                              | Cancels inline edit; collapses revealed actions; closes open slider; cancels in-flight drag.                                                                     |

**Regression-guard rules (BDD scenario must assert):**

- Mouse `pointermove` over any row WITHOUT a click MUST NOT reveal action chips, MUST NOT change row background, MUST NOT show pen icon.
- Double-click on a category header cell (name / planned / cushion) MUST NOT enter inline edit; MUST be a no-op or — if Claude's discretion picks — also reveal options.
- `pointermove` over the grid MUST NOT highlight columns, rows, or cells.

## Accessibility Contract

- **Keyboard:** Every interactive surface reachable via Tab. Drag-reorder accessible via `Space` to lift, arrow-keys to move, `Space` to drop (dnd-kit native keyboard sensor).
- **Screen reader:** All revealed action chips labeled (`aria-label`); dnd-kit `screenReaderInstructions` configured per a11y docs. Inline-edit announces "editing amount, press Enter to save, Escape to cancel".
- **Focus management:** Slider open → focus moves to first input. Slider close → focus returns to triggering element. Inline-edit-end → focus returns to the row.
- **Live regions:** Month change announced via `aria-live="polite"`; optimistic-row state transitions announced ("transaction saving", "transaction saved", "save failed, retry available").
- **Color contrast:** All text/background pairs verified against WCAG AA 4.5:1. Yellow `--primary` on `--canvas-dark` = 14:1; `--destructive` on `--canvas-dark` = 5.2:1; `--body-on-dark` on `--canvas-dark` = 13:1; `--muted-foreground` on `--canvas-dark` = 4.6:1 (passes AA).
- **No hover-only behavior:** Per D-PH4-INT1 — hover does nothing. Eliminates the "hover-discoverable-only" a11y problem.
- **Touch targets:** Mobile minima per spacing table above (44×44 via outer padding where visible glyph is smaller).
- **Reduced motion:** All transitions (drag animations, slider slide-in, optimistic fades) respect `prefers-reduced-motion: reduce` and collapse to instantaneous state changes.
