# Phase 03 — Budget Restructure (SPEC)

**Status:** spec captured 2026-05-11 from design-review brainstorming. Not yet planned by `/gsd-new-milestone`. This file is the durable input to the planning workflow.

**One-line:** Replace multi-page workspace UI with a single Excel-like Budget Detail Page (BDP) keyed by URL; rename `workspace`→`budget` and `account`→`wallet`; drop income/transfer kinds; reserves auto-compute; cushion is a workspace-wide toggle; nuke dev DB.

## 2. Information architecture

### Top navigation

- **Logo / app name** (links to `/`)
- **Budget switcher dropdown** — name of current budget + private/shared icon + chevron. Click opens dropdown grouped by `Personal` / `Shared`. `+` button is rendered **aside** the dropdown trigger (not as a list item) — opens new-budget wizard.
- **User menu** (right side; sign-out, settings, locale).

### Routes

| Route                                                      | Purpose                                                                                                            |
| ---------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------ |
| `/`                                                        | Home: combined view (cards per budget + placeholder chart)                                                         |
| `/budgets/new`                                             | Onboarding wizard for first budget (also reachable from `+` button)                                                |
| `/budgets/[id]/spendings`                                  | Excel-like categories grid (default tab)                                                                           |
| `/budgets/[id]/reserves`                                   | Per-category reserve balances + reserve wallet reconciliation                                                      |
| `/budgets/[id]/wallets`                                    | Inline-editable wallet rows                                                                                        |

| `/sign-in`, `/sign-up`, `/verify-email`, `/reset-password` | Unchanged from phase 02 (Better Auth)                                                                              |

### BDP tabs

Pill-style horizontal tabs at top of BDP, **sticky on scroll**. Order: Spendings · Reserves · Wallets · Settings. Active pill highlighted with the single yellow accent (per DESIGN.md). Above tabs: a **Task banner** when tasks exist (`N tasks` chip, expandable into list).

## 3. Spendings tab — the core grid

Excel-like layout, **columns = categories**, current month only. Arrow keys ←/→ navigate between months without leaving the tab.

### Column header (top rows of each column)

| Row | Content                                                        | Editable                       |
| --- | -------------------------------------------------------------- | ------------------------------ |
| 1   | Category name + (on hover) pen icon                            | Pen → side slider              |
| 2   | Planned (or Cushion when toggle on)                            | Read-only here; edit in slider |
| 3   | Overspent (computed: `max(0, spent − planned − reserve_used)`) | No                             |
| 4   | Reserves used this month (computed)                            | No                             |
| 5   | Balance (computed: `planned + reserve_used − spent`)           | No                             |

Below header, the column shows the list of this-month transactions for that category, newest first. Bottom of each column has a **quick-entry input**: type a number → Enter → creates txn with `(date=today, category=this column, amount=value, currency=budget.currency, note=null)`.

### Cell behaviors

- **Empty quick-entry input**: type `5.96` + Enter → txn saved, list refreshes, input clears.
- **Hover a txn row**: pen icon appears → opens side slider for full edit (date, category, amount, currency override, note). Side slider also has delete.
- **Recurring drafts**: rendered as **highlighted rows** (distinct background per DESIGN.md — likely amber/warning tone) with an explicit "Confirm" affordance. Click → becomes real txn (uses rule's defaults). User can also edit before confirming.
- **No search bar, no filter chips** — current-month-only scope keeps grid clean.

### Add category

A **dashed `+` column** at the rightmost edge of the grid. Click → same side slider as pen icon (in "create" mode). Slider fields: name, planned value, cushion value, monthly reserve % (optional? — see §4), icon/color (optional, low priority).

### Reorder

Drag-to-reorder by column header. Order persisted per-budget (`category.sort_index INTEGER`).

### Month navigation

- Above grid (or in tab pill row): current month label + `←` and `→` chevrons.
- Arrow keys ←/→ at root level shift month back/forward.
- Past months show same grid, read-only quick-entry input (still allow pen-edit on existing txns).
- Future months not browseable (or browseable but no quick-entry).

### Mobile

Same grid, **horizontal scroll**. Pinch-zoom retained. Mobile-first PWA still ships, but Spendings tab specifically deprioritizes touch ergonomics in favor of grid parity.

## 4. Reserves tab

**Table** with columns:

| Category | Reserve balance | Reserve wallet share         | Actions |
| -------- | --------------- | ---------------------------- | ------- |
| Housing  | 300.00          | (computed: proportional sum) | —       |

- **Reserve balance** is auto-computed: cumulative `(active_budget − spent)` over past months for that category, floor 0, minus any amount pulled to cover overspend in subsequent months. Per-category isolated — Housing reserve cannot fund Groceries overspend.
- "Active budget" = `planned` when cushion mode is off, `cushion` when on. Computed per-month using the active mode **at that historical month**.
- Reserve wallet share = (this category's reserve / sum of all reserves) × (sum of reserve-type wallet balances). Used to detect mismatch.

### Reconciliation task

A **Task** of kind `RESERVE_TOPUP` is generated when `Σ(category reserve balances)` ≠ `Σ(reserve-type wallet amounts)`. Task title (i18n): `"Top up reserve by {amount}"` (or `"Withdraw {amount} from reserve"` if negative). No prescribed source wallet — user manually adjusts wallet balances to reconcile. Task auto-resolves when sums match.

## 5. Wallets tab

Inline-editable rows. Columns:

| Name | Currency | Amount | Type | Cushion checkbox? |
| ---- | -------- | ------ | ---- | ----------------- |

Wait — clarification: user spec'd "checkbox if it's a cushion wallet" but also "three types: spendings/cushion/reserve". Reconcile: **Type is a single-select** (radio or segmented control inside the row): Spendings / Cushion / Reserve. No separate checkbox.

- All cells directly editable in place (text/numeric inputs). Tab moves between cells.
- Auto-save on blur. Toast confirmation.
- `+ Add wallet` row at bottom (clicking spawns a blank row with focus on name).
- No popup. No pen icon. No archive (delete is row-level via trash icon on hover).
- Type is **display label only** — no behavioral difference except sums on home/reserves/dashboard. No income tracking, no transfer ledger.

## 6. Settings tab

Sections (vertically stacked):

1. **Budget identity** — name (editable), currency (editable until first txn, then locked + read-only with tooltip).
2. **Cushion mode** — single toggle. Persists `budget.cushion_mode_enabled boolean`. When on: all grid headers display `cushion` instead of `planned`; all reserves/overspent calc uses cushion values.
3. **Recurring rules** — CRUD list. Cadence: daily / weekly / monthly / yearly with day-of-\* selectors. Each rule: name, amount, currency, category, cadence, start date, end date (optional), active toggle. Drafts auto-generate via pg-boss when due → appear as highlighted rows in Spendings grid.
4. **Members** (only when `budget.kind = SHARED`) — list members, generate share link (token-based invite via Better Auth orgs plugin), revoke member, leave budget.
5. **Danger zone** — archive budget (soft-delete), delete budget (hard-delete with confirm-typed-name).

Categories are NOT directly managed here — they're managed via pen-icon on Spendings tab. (Decision: keep settings minimal; categories live where they're used.)

## 7. Transactions

### Schema (after restructure)

```ts
transactions {
  id: uuid pk,
  tenant_id: uuid, // RLS
  budget_id: uuid fk,
  category_id: uuid fk,
  date: date,
  amount_original_cents: bigint,
  currency_original: char(3),
  amount_converted_cents: bigint, // in budget.currency
  fx_rate: numeric(19,8) | null, // null when currency_original == budget.currency
  fx_as_of: date | null,
  note: text | null,
  recurring_rule_id: uuid fk | null, // set when materialized from a rule
  confirmed_at: timestamptz | null, // null = draft pending confirm; set when user confirms
  created_at, updated_at, deleted_at,
}
```

**Dropped columns from phase 02:** `kind`, `account_id`, `to_account_id`, `direction`.

### Edit slider fields

Date · Category · Amount · Currency (default budget; dropdown to override) · Note. Delete button.

### FX

- Convert at transaction date using `FxProvider` port (Frankfurter adapter).
- Store both original and converted amounts.
- Grid always displays converted (budget currency).
- Side slider shows both: "5.00 USD · ~4.20 EUR @ 0.84 (2026-05-11)".

## 8. Reserves auto-compute algorithm

For category `c` at end of month `m`:

```
active_budget(c, m) = c.cushion if budget.cushion_mode_was_on_at(m) else c.planned
remainder(c, m)    = max(0, active_budget(c, m) − Σ(txns in c during m))
reserve_pulled(c, m) = max(0, Σ(txns in c during m) − active_budget(c, m) − previous_reserve(c, m-1))
reserve(c, m)      = max(0, reserve(c, m-1) + remainder(c, m) − reserve_pulled(c, m))
```

Edge cases:

- New category (no history): `reserve = 0`.
- Cushion toggle flipped mid-year: each historical month uses the mode-as-of-that-month (requires tracking `cushion_mode_history` — likely a SCD-2 mini-table or a snapshot in audit log).

Implementation choice (defer to planning phase): probably a materialized view refreshed by trigger on `transactions` and `category_limits` writes, OR computed on read via a view/function.

## 9. Tasks queue

**v1.1 rescope (2026-05-30):** Tasks queue ships 3 kinds — RESERVE_TOPUP, CONFIRM_DRAFT, CUSHION_BELOW_TARGET. STALE_WALLET and MONTH_END_REVIEW dropped.

| Trigger                                                                                                          | Title (i18n key)             | Resolution                               |
| ---------------------------------------------------------------------------------------------------------------- | ---------------------------- | ---------------------------------------- |
| `Σ reserves ≠ Σ reserve-type wallets`                                                                            | `tasks.reserve_topup`        | Sums match                               |
| Recurring rule due → draft created                                                                               | `tasks.confirm_draft`        | User confirms draft                      |
| `cushion_enabled = true AND Σ(category.cushion × cushion_target_months) > Σ(CUSHION wallets FX→budget currency)` | `tasks.cushion_below_target` | shortfall ≤ 0 OR cushion_enabled = false |

Storage: `tasks(id, tenant_id, budget_id, kind, payload_json, status, created_at, resolved_at)`.

Surface: BDP top banner. Chip: `N tasks pending`. Click expands inline list with primary action button per task.

Web-push notification fires on task create (per user notification preferences).

## 11. Onboarding wizard

After signup (and when user clicks `+` button in switcher dropdown):

1. Budget name input.
2. Currency picker (default = browser locale guess).
3. Budget type: Private (default) or Shared.
4. Starter category template (multi-select): Housing, Groceries, Transport, Eating Out, Entertainment, Health, Subscriptions, Other. Each pre-populates planned + cushion as 0 (user edits later).
5. Optional: Skip → empty budget.

On finish: redirect to `/budgets/[new_id]/spendings`.

Onboarding state stored in `onboarding_progress(user_id, step, completed_at)` so wizard is resumable.

## 14. Migration approach

**Nuke dev DB.** No data preservation. Steps:

1. New Drizzle migration `2026-05-xx_phase03_restructure.sql`:
   - DROP tables: `transactions`, `accounts`, `workspace_members` (if exists outside Better Auth orgs)
   - DROP / RENAME: `workspaces` → `budgets` (or DROP+CREATE)
   - CREATE new `wallets`, `transactions` (per new schema), `tasks`, etc.
2. CI gate updated: tenant-leak tests target new table names.
3. E2E rewritten for new flows (workspaces→budgets, accounts→wallets, no transfer/income/account fields).

Acceptable because no production deployment.

## 17. Success criteria

- New user signs up, completes onboarding wizard, lands on `/budgets/[id]/spendings` with starter categories.
- User adds a txn via quick-entry input — appears in grid in < 200ms.
- User edits planned + cushion via pen icon — values version per SCD-2.
- User enables cushion mode — all displayed values switch to cushion; reserves recompute.
- User overspends Housing — system pulls from Housing reserve real-time; "Reserves" tab shows updated balance.
- User updates wallet balance manually → reserve mismatch resolves → task auto-completes.
- Recurring rule due → draft row highlighted in Spendings grid → user clicks Confirm → real txn.
- Wallet idle 30 days → stale task fires → push notification delivered.
- All flows green in playwright-bdd. Zero TypeScript errors. Tenant-leak CI gate green on new tables. 80% domain coverage in `bunfig.toml`.
