<next_scroll_timing_note>
BdpTabs uses `<NavLink>` -> next/link with default `scroll` (undefined => Next default
scroll-to-top on push). NavLink (nav-link.tsx) does NOT pass `scroll={false}`, so Next SHOULD
scroll window to top on forward nav — but on iOS WebKit App Router this is unreliable for
same-layout-segment client nav, and any scroll-restoration on back/forward can run AFTER a
mount effect. We therefore do NOT rely on Next: we reset explicitly, keyed on pathname, inside
a layout effect deferred one rAF so it wins over late restoration. Both writes are idempotent.
</next_scroll_timing_note>
