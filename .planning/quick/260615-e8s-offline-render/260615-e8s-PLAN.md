<objective>
Make the Budget PWA offline experience actually work: four device-confirmed fixes shipped as atomic, independently-committable tasks on branch `tasks-redesign`.

The offline data layer is dead code: `cacheBudgetSnapshot()` (the only IDB + sync-meta writer) is called only by `useBudgetData`, which is never mounted; and there is no read-back path (`getCachedBudget` has zero consumers). So IDB stays empty → cache-age "unknown" + blank offline reload. Plus two indicator polish fixes (icon swap, tap-to-close race).

Purpose: offline = read-only-reliable. A previously-visited route must paint its last-online data + chrome instantly offline; the indicator must show a real cache age and close on tap.

Output: indicator (Unplug + Popover), populated IDB cache + sync-meta on every online visit, an offline read-back path feeding cached rows into the rendered queries, and an offline-renderable home list.

Robust-minimal constraint: DO NOT reintroduce the offline write-queue / replay / sync-issues (deliberately removed). Read-only offline only.
</objective>
