<chosen_variant>
**Box height — measured-top + screen-extension (extends the existing effect, not a new mechanism).**
`height = max(160px, calc(100lvh - ${top}px + ${extension}px))`
where `extension = computeScreenExtension({screenH, lvhPx, isCoarsePointer, isIOS})`.

- `lvhPx`: one-shot probe element `position:fixed; height:100lvh` → `getBoundingClientRect().height` (round). Deterministic, matches the lvh bottom anchor. Probe created+removed inside the effect (mirrors `probeEnvBottom` :307-315).
- `screenH`: PORTRAIT long-edge. iOS swaps `screen.width`/`screen.height` per orientation, so resolve as:
  `portrait = matchMedia('(orientation: portrait)').matches; screenH = portrait ? screen.height : screen.width`.
  (Picks the dimension that is the device's portrait height regardless of current orientation reporting.)
- Why ADD to lvh (not replace with screenH−top): preserves the measured-top self-correction for every band above the grid (the whole point of R15's architecture). extension is purely the lvh→screen delta.

**Spacer — DYNAMIC via CSS var, NOT a static floor.**
Rejected static ~150px: e2e upper bound is `gap <= 128` (steps.ts:387) and Chromium matches `display-mode: browser`, so a static 150 would FAIL Chromium. Dynamic wins:

- Effect writes `--grid-tail-spacer-h: calc(env(safe-area-inset-bottom,0px) + ${96 + extension}px)`.
- Chromium / desktop / Android: extension == 0 → `env+96` → identical to R15/R16, e2e `gap` stays at 96 (<=128). PASS.
- iOS browser bar-shown: extension ≈ 40 (844−754=90 lvh→screen, but lvhPx is measured with bar collapsed state of lvh = 754; extension clamps within [0,140]) → spacer ≈ env+136 → last row clears the VISIBLE viewport (vvH 714). Device-only, not e2e-gated.
- Spacer var is recomputed in the SAME `updateMaxH` (so it respects the keyboard freeze automatically — no extra listener).
- global.css browser rule changes from a literal `+96px` to consuming the var with a `96px` fallback:
  `[data-grid-tail-spacer]{height: var(--grid-tail-spacer-h, calc(env(safe-area-inset-bottom,0px)+96px));}`.
- Standalone JSX fallback (`env+64`) and the standalone @media block stay UNTOUCHED — the var is only written/consumed in browser mode (standalone extension==0 would write env+96, but standalone never sets the var because we only setProperty when `!isStandalone`; the JSX `+64` fallback class governs standalone). Keep standalone bit-identical.

**Gate (the load-bearing safety):** `computeScreenExtension` returns 0 unless `isIOS && isCoarsePointer`. Even on iOS it clamps `screenH − lvhPx` to `[0, 140]`. Desktop screen.height (e.g. 1440) − viewport (e.g. 800) = 640 → would blow layout, but desktop fails the `isCoarsePointer && isIOS` gate → 0. Android (coarse pointer, not iOS) → 0 (Android Chrome already composites correctly; do not touch). Belt: even if a future caller passes a bad combo, the [0,140] clamp caps blast radius at 140px.
</chosen_variant>
