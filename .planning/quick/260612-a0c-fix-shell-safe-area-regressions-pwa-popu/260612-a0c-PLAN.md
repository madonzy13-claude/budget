<objective>
Fix two regressions introduced by the SHELL-R8..R11 iOS safe-area work (commits 153bead..407e677), with root-cause analysis and automated geometry proof BEFORE any manual device test.

Regression 1 (standalone PWA): edit-category / edit-transaction / edit-recurring-rule sheets render shifted up with a gap below them. All three are `side="right"` Radix sheets (`inset-y-0 right-0 h-full`) — full-viewport-height drawers, NOT bottom sheets. They portal to `document.body`. The `.pb-shell-safe` standalone padding (`calc(env(safe-area-inset-bottom) + 64px)`) lives INSIDE the scrolling page content and must stay (it is the only iOS-WebKit-honored bottom-clearance mechanism). The sheet's bottom edge must reach the true visual-viewport bottom (home indicator), decoupled from page padding.

Regression 2 (Safari browser mode): on the BDP, the tasks banner (`pill-task-slider`, rendered inside `[data-bdp-tabs]` sticky wrapper via BdpTabs) is half-hidden behind the pinned `[data-shell-header]`. The header is `position: sticky; top: 0` in browser mode; `[data-bdp-tabs]` is offset to `top: calc(4rem + 1px)`. The banner sits inside that wrapper but its own offset/z-index lets it slide under the header during native page scroll.

Purpose: restore correct popup + banner geometry on real iOS while preserving every hard-won shell behavior.
Output: decoupled Sheet positioning, banner offset fix, updated regression suite, new browser-mode E2E geometry scenario, live Playwright proof.
</objective>

HARD-WON CONTEXT — do not rediscover:

- iOS WebKit ≠ Chromium for this shell. Scroll-container padding-bottom ignored; ::after spacers never enter scroll flow; only IN-FLOW page-content padding (.pb-shell-safe) extends scrollHeight on iOS. That is WHY .pb-shell-safe exists. Do NOT replace the mechanism — scope its blast radius.
- .pb-shell-safe is standalone-only (`@media (display-mode: standalone)`), on the BDP children wrapper + home page root. KEEP IT.
- [data-shell-header] is `position: sticky; top: 0` in `@media (display-mode: browser)` so iOS Safari's bottom bar collapses on native page scroll. KEEP IT.
- [data-bdp-tabs] is offset `top: calc(4rem + 1px)` in browser mode (h-16 header + 1px border). KEEP IT.
- Shell outer wrapper has `filter: var(--ptr-filter, none)`. A non-`none` filter creates a containing block for `position: fixed` descendants (known dnd-kit ghost bug — line 200-208 of (app)/layout.tsx). At rest --ptr-filter is `none`, so fixed = ICB. Sheets portal to document.body (outside that wrapper) so they are NOT affected by the filter. The standalone gap is the home-indicator inset, NOT the page padding directly.
- Headless Chromium gives FALSE PASS on device-only iOS issues. Playwright projects are chromium + mobile (both Desktop-Chrome-based) — NO webkit, and `display-mode: standalone` is NOT Playwright-emulatable. Therefore: standalone (Regression 1) is guarded by Vitest geometry/matchMedia-stub + source assertions; browser-mode (Regression 2) is fully provable by Playwright geometry (banner top vs header bottom). Tests MUST assert geometry (bounding boxes vs viewport), not just CSS-rule presence, wherever measurable.
  </root_cause_notes>
