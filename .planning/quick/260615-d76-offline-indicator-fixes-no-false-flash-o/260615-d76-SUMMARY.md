## The four fixes + how each is verified

### Fix 1 — No false flash on reload

- `isOnline` now inits `true` (was `navigator.onLine`). On reload `navigator.onLine`
  is briefly false during load → old code flashed the indicator even when online.
- A post-mount `useEffect` now reads the REAL `navigator.onLine` and attaches the
  online/offline listeners. Indicator appears only after a confirmed post-mount
  `navigator.onLine===false` OR an `offline` event.
- Tests: "no false flash: online machine never shows the indicator after mount"
  - "post-mount confirmed offline … shows the indicator" (components test). GREEN.

### Fix 2 — Icon → crossed cloud (CloudOff)

- Replaced lucide `Globe` with `CloudOff`. Kept `animate-pulse`, red
  `--destructive`, and the h-6 sr-only-online / inline-offline zero-height
  envelope (no layout shift). testid `offline-globe` → `offline-cloud-off`.
- Tests: "renders a pulsing CloudOff icon" asserts `lucide-cloud-off` +
  `animate-pulse` and that no `offline-globe` exists. Layout-invariant test file
  updated to `offline-cloud-off`. GREEN.
- Served: `offline-cloud-off` present (1 file), `lucide-globe`/`offline-globe`
  absent (0 files) in the built `(app)/layout` chunk.

### Fix 3 — Home/budget-list shows real cache age (not "unknown")

- `offline-cache.ts`: added `getMostRecentSyncMeta()` — `getAll("sync-meta")`,
  max `lastSyncedAt` across ALL rows (ISO-8601 UTC strings compare
  lexicographically; includes `__global__`). Returns null only if nothing synced.
- `use-cache-on-fetch.ts`: `cacheBudgetSnapshot` now ALSO writes
  `setSyncMeta("__global__", iso)` alongside the per-budget key — every cache
  write bumps a global timestamp.
- Badge cache-age fallback chain: per-budget `getSyncMeta(budgetId)` →
  `getSyncMeta("__global__")` → `getMostRecentSyncMeta()` → only then
  `tooltipUnknown`. budgetId-null route now shows a real "data from N ago".
- Tests: `getMostRecentSyncMeta` (null / newest across rows / includes
  `__global__`); the `__global__` write in use-cache-on-fetch; badge "null
  budgetId falls back to global", "per-budget miss → global → most-recent". GREEN.
- Read-cache scope unchanged — only the sync-meta AGE source is global.

### Fix 4 — Tooltip below icon + tap-to-close toggle

- `TooltipContent side="bottom"`.
- Trigger `onClick={() => setOpen((o) => !o)}` — controlled `open` is the source
  of truth, so tap opens then a second tap reliably CLOSES (no reopen flicker);
  desktop hover/focus still driven by Radix `onOpenChange`.
- Tests: "tooltip content renders with side=bottom"; "tap-to-close: … toggles
  open then closed" (open→true→false). GREEN.
- Served: `"bottom"` present in component chunk.
