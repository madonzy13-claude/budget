## New write-path design (T1)

- `onMutate`: unchanged optimistic insert, BUT optimistic row drops `pending`/`unsent`
  flags (no in-flight pending UI for offline). `ctx = { previous, optimisticId }`.
- `mutationFn`: ALWAYS attempt the POST (no `navigator.onLine` fast path).
  Keep `AbortSignal.timeout(8000)` so a dead link fails fast. Keep explicit
  `X-Budget-ID` header (harmless). Classify failure:
  - network throw (`TypeError`) / `AbortError` (timeout) / `res.status >= 500`
    → `throw new OfflineWriteError()` (replaces `OfflineEnqueuedError`; NO enqueue).
  - `4xx` → `throw new Error(await res.text())` (genuine error).
  - ok → return server row (unchanged onSuccess swap).
- `onError(err, _input, ctx)`: **ROLL BACK** — restore `ctx.previous` for
  `["transactions",budgetId,month]` and re-invalidate `["spendings-summary",budgetId]`
  (the optimistic spent bump must also revert; simplest: invalidate summary so the
  engine value returns). Then toast:
  - `err instanceof OfflineWriteError` → `toast.error(t("write.offline"))` (new string).
  - else → `toast.error(t("write.failed"))` (generic) — or existing grid error string.
  - Capture `const t = useTranslations("grid.txn")` (or a `sync`→repurposed ns)
    at hook top so onError closure has it.
- `onSettled`: keep the invalidations (harmless on rollback; authoritative refetch).

## New SW navigation strategy (T3) — cached page offline, no stuck screen

Current: navigation fail → STATIC `/offline.html` (a 503 doc) — the stuck screen.
New `handleNavigationRequest(request, fetchFn, matchCache, makeInlineNotice)`:

1. Try `fetchFn(request)` with the existing 5s timeout.
2. `res.status >= 500` → fall through to step 4 (treat as unreachable).
3. ok (2xx/3xx/4xx<500) → return `res` (online path + redirects unchanged).
4. On throw OR 5xx: `const cached = await matchCache(request)` — return the CACHED
   navigation document for this route if present (visited routes render offline).
   - Serwist precaches the app shell + visited pages land in the runtime "pages"
     cache via navigationPreload/precache; match by request URL, `ignoreSearch`.
5. Cache MISS → return `makeInlineNotice(request)`: a MINIMAL, self-recovering
   inline 503 (localized title from `SUPPORTED_LOCALES`) with `online`/`focus`/
   `visibilitychange`→`location.reload()` auto-recovery and NO `/api/health`
   gate — never a stuck full-page doc requiring app restart.
   Reuse the locale logic; drop `decideOfflineRecovery`/`sanitizeNext`/the
   3-probe health gate entirely.

- sw.ts: SW MUST still `skipWaiting:true` + `clientsClaim:true` + keep the
  `/api/*` NetworkOnly denylist + style/script caching + activate-purge + the
  notificationclick handler — only the navigation branch changes.
